### v1 → v2 변경사항
- Undo 스낵바: 미구현 → **구현 완료** (`UndoSnackbar` + 5초 카운트다운 + 호버 일시정지)

- 올데이 이벤트 행: 부분 구현 → **구현 완료** (일간/주간뷰 모두 지원)

## 5. 비주얼 디자인

## 9. 카테고리별 상세 피드백

### 9.1 레이아웃 & 구조

**강점:**
- `TopBarCenterSlotPortal` 패턴으로 전역 탑바에 캘린더 컨트롤을 자연스럽게 통합
- 사이드바 6개 섹션(생성/미니캘린더/탭/필터/미완료/더보기)은 도메인에 맞게 풍부
- `CalendarLayoutShell`의 모바일 오버레이 사이드바는 적절한 반응형 처리

**개선 필요:**
1. **사이드바 정보 과밀** — 6개 섹션이 동시에 보이면 인지 부하 높음. Google처럼 접을 수 있는 아코디언 또는 우선순위별 숨김 처리 권장
2. **주간뷰 모바일 가독성** — 7컬럼이 모바일에서 너무 좁음. `<768px`에서 3일뷰 자동 전환 또는 일간뷰로 전환 필요
3. **미니 캘린더 ↔ 메인뷰 연동** — 미니 캘린더의 날짜 선택 범위가 메인뷰에서 하이라이트되어야 함 (Google 패턴)

### 9.2 인터랙션 & 조작감

**강점:**
- `useCrossDayDrag`의 200ms 롱프레스가 클릭/드래그 정확히 구분
- `useDragToCreate`의 5px 최소 이동 임계값이 클릭/드래그 구분에 효과적
- `useResizable`의 rAF 기반으로 매끄러운 리사이즈
- 15분 스냅 그리드는 Google과 동일한 표준
- Undo 스낵바의 호버 일시정지는 훌륭한 UX 터치

**개선 필요:**
1. **드래그 고스트 블록 디자인** — 파란 점선 사각형 대신 실제 블록의 그림자 복사본이 더 직관적. 소스 위치에 투명한 원래 모양 플레이스홀더 표시
2. **더블클릭 → 전체 편집** — Google은 이벤트 더블클릭 → 풀페이지 편집. 현재는 클릭 → 팝오버만. 더블클릭으로 모달/슬라이드오버 편집 폼 열기 고려
3. **Undo 단일 레벨 제한** — 현재 새 액션이 이전 액션을 덮어씀. 연속 실수 시 마지막 것만 복구 가능. 최소 3~5레벨 스택 고려
4. **드래그투크리에이트 주간뷰** — 구현되어 있으나, 크로스데이 드래그와의 상호작용 명확한지 확인 필요 (같은 마우스 제스처가 충돌하지 않는지)

### 9.3 비주얼 & 시각 피드백

**강점:**
- 과목별 해시 기반 자동 색상이 일관성 유지에 효과적
- 3단계 밀도 레벨(short/medium/long)이 시간에 비례한 정보량 표시에 적절
- 상태별 시각 처리(완료/연기/미수행)가 교육 도메인에 최적화
- 충돌 표시(오렌지 보더)는 Google에 없는 유용한 기능
- `HoverPreviewCard`의 300ms 딜레이 + 터치 디바이스 자동 비활성화가 정교함

**개선 필요:**
1. **이벤트 블록 시각적 폴리시** — `rounded-lg` → `rounded-xl`, 미묘한 그림자 계층(`shadow-sm` + 호버 시 `shadow-md`), 호버 시 자연스러운 엘리베이션 변화로 MD3 느낌 강화
2. **비학습시간 대각선 패턴** → 단색 배경 + 타입 아이콘(🏫📖💤🍽)으로 시각적 노이즈 감소. Google "Out of Office" 블록 참고
3. **완료 블록 60% 투명도 과함** — 정보 확인이 어려움. 80% 투명도 + 좌측 체크마크로 완료 표시가 가독성 유지에 유리
4. **로딩 상태** — 주간뷰 28개 쿼리 로딩 중 빈 화면 대신 캘린더 그리드 스켈레톤 표시. 이벤트 블록 위치에 시머 효과

### 9.4 정보 구조 & 네비게이션

**강점:**
- 사이드바 탭(Planner/Settings/Analytics/Progress/History)은 분석/히스토리 기능 제공
- 미완료 플랜 섹션은 GTD "미결 항목" 개념 반영
- `EventPopover`의 진행바가 학습 진도를 직관적으로 표시
- 사이드바 검색의 디바운스 + 노란 하이라이트가 효과적

**개선 필요:**
1. **사이드바 미완료 플랜 20개 제한** — "가장 오래된 5개" + "전체 보기" 링크가 더 나은 패턴
2. **필터 활성 상태 표시** — 검색은 노란 보더로 활성 표시됨. 그룹/콘텐츠 타입 필터도 활성 시 시각적 표시 필요
3. **검색 결과 네비게이션** — 현재 검색 매칭 블록이 하이라이트되지만, "다음 결과로 이동"/"이전 결과로 이동" 기능 없음. 검색 결과가 여러 날에 분산될 때 유용

## 11. 결론

### 전체 평가

**달성률 ~65%** — v1(55%) 대비 10%p 향상. Undo 시스템, 검색, 호버 프리뷰, 올데이 이벤트 등 핵심 갭이 해소되면서 기본 사용성은 Google Calendar 수준에 근접.

### 현재 가장 큰 갭 3가지

1. **월간뷰 이벤트 칩 미표시 (P1)** — 월간 개요에서 "이 날에 뭐가 있는지" 즉시 파악 불가. Google은 색상 칩 + 제목으로 개요 제공
2. **드래그 시각 피드백 조잡함 (P1)** — 파란 점선 고스트 대신 실제 블록 복사본 + 투명 플레이스홀더가 필요. "이동 중" 느낌의 품질 차이
3. **모바일 최적화 부족 (P1)** — 주간뷰 7컬럼이 모바일에서 사용 불가 수준. 3일뷰 자동 전환 또는 모바일 전용 레이아웃 필요

### 현재 가장 큰 강점 3가지

1. **AI 통합 (생성/최적화)** — 캘린더에서 직접 AI 호출은 혁신적
2. **학습 상태 시각 시스템** — 진행률/완료/연기/미수행의 직관적 시각 표현은 교육 도메인 필수
3. **Undo 시스템 완성도** — 5가지 액션 타입 + 호버 일시정지 + 카운트다운 진행바는 Google 수준 이상

### 다음 단계 권장

1. **P1 항목 4개** (월간 칩, 드래그 피드백, 모바일 최적화, 스켈레톤 로딩) 완료 시 → **달성률 ~75%** 예상
2. **P2 상위 5개** (뷰 전환 트랜지션, 날짜 슬라이드, 색상 선택, 비학습시간 리디자인, MD3 폴리시) 완료 시 → **달성률 ~85%** 예상
3. 85%+ 수준이면 Google Calendar 대비 "도메인 특화 + 동등한 사용성"으로 경쟁력 확보
