## 1. 적대적 리뷰 요약

### 1.1 컨설턴트 시각 — "현장을 모르는 설계"

| # | 비판 | 심각도 | 핵심 |
|---|------|--------|------|
| C1 | 컨설턴트는 그래프가 아닌 **서사(내러티브)**로 사고 | CRITICAL | 노드-엣지는 분석 도구, 작업 도구 아님 |
| C2 | 4-레이어가 맞지 않는 영역 다수 (행특, 독서, 자율활동) | HIGH | 가이드 레이어가 불필요한 영역에 강제 |
| C3 | 📝가안은 레이어가 아니라 **메모/대화** | HIGH | 구조화된 데이터가 아닌 암묵지 |
| C4 | AI 추천 주제가 같은 학교 학생끼리 중복될 위험 | CRITICAL | 입학사정관이 "컨설팅 티" 감지 |
| C5 | 학부모에게 그래프를 보여주면 해약 | HIGH | 학부모는 "잘 되고 있나요" + 다음 액션만 원함 |
| C6 | 학생은 앱을 안 씀. 투두 리스트만 필요 | MEDIUM | 탐구 가이드 앱을 쓰는 학생은 5% 미만 |
| C7 | AI가 컨설턴트를 대체하는 것으로 보이면 저항 | CRITICAL | 시스템 = 컨설턴트를 효율적으로 만드는 도구여야 함 |
| C8 | 완벽한 일관성은 오히려 의심받음 (80%가 자연스러움) | HIGH | "불완전함 보존" 원칙 필요 |
| C9 | 학생 8명으로 그래프는 과잉 — 엑셀이면 충분 | HIGH | 데이터 밀도가 충분해진 후 (학생 50+, 3년치) |

### 1.2 프론트엔드 아키텍트 시각 — "Second System Effect"

| # | 비판 | 심각도 | 핵심 |
|---|------|--------|------|
| F1 | 100 노드 × 4 레이어 = 400 쿼리 또는 거대한 재분배 | CRITICAL | 현재 9 쿼리가 이미 최적 |
| F2 | 노드별 5 상태 × 100 = 500 상태 변수 — 버그 팩토리 | CRITICAL | AI 완료 + 편집 중 레이스 컨디션 |
| F3 | 현재 배치 쿼리 아키텍처가 이미 정답 | HIGH | 파괴하면 안 됨 |
| F4 | 상태 적응형 렌더링 = 조합 폭발 (2⁴×3역할 = 48 변형) | HIGH | 테스트 지옥 |
| F5 | 모바일에서 100개 카드 = 사용 불가 | HIGH | 현재 선형 스크롤이 모바일에 더 적합 |
| F6 | Cmd+F 검색이 카드 접기에 의해 깨짐 | MEDIUM | 현재 전체 스크롤은 브라우저 검색 가능 |
| F7 | 11~19주 리라이트 vs 2.5주 점진 개선 | CRITICAL | 8명 학생에게 19주 투자는 ROI 없음 |
| F8 | StudentRecordClient 1,295줄은 리팩토링이 필요하지, 리라이트가 필요한 게 아님 | HIGH | 4개 스테이지 서브컴포넌트 추출로 충분 |

### 1.3 정보설계 전문가 시각 — "문서 메타포를 파괴하면 안 됨"

| # | 비판 | 심각도 | 핵심 |
|---|------|--------|------|
| I1 | 컨설턴트=문서, 학부모=리포트, 학생=투두 — 그래프는 누구의 멘탈 모델도 아님 | CRITICAL | 그래프를 강제하면 3역할 모두 불편 |
| I2 | 현재 TOC+스크롤이 O(1) 인지 비용, 그래프는 O(n) | HIGH | 익숙한 NEIS 형식이 네비게이션 최강 |
| I3 | 생기부 법적 위계(sec 1~9)를 그래프가 평탄화 | HIGH | sec-7(세특)이 sec-2(출결)과 동일 무게 |
| I4 | 엣지 7종 × 100노드 = 스파게티 | MEDIUM | 전체 시각화보다 on-demand 1-hop |
| I5 | 레이어가 순차적이지 않음 — 순환적 | MEDIUM | 분석→가이드 역행 빈번 |
| I6 | 빈 그래프는 영감이 아니라 절망 | HIGH | 학부모가 보면 "돈 내고 아무것도 안 했네" |
| I7 | 3개 뷰(편집/프레젠테이션/탐색)를 하나의 UI에 넣으면 모두 타협 | CRITICAL | 명시적 3모드가 현실적 |

## 2. 비판에서 도출된 핵심 원칙

### 합의된 원칙 (3명 전원 동의)

| # | 원칙 | 근거 |
|---|------|------|
| **P1** | **문서 메타포를 유지하라** | 현재 NEIS 레이아웃이 컨설턴트의 멘탈 모델 |
| **P2** | **그래프는 분석 도구, 작업 도구 아님** | 편집은 문서에서, 관계 탐색은 보조 패널에서 |
| **P3** | **기존 아키텍처를 파괴하지 마라** | 9쿼리 배치 + TOC 스크롤 + 자동저장이 이미 최적 |
| **P4** | **점진적 개선 > 전면 리라이트** | 2.5주 80/20 vs 19주 풀 리라이트 |
| **P5** | **AI = 검증 보조, 대체가 아님** | 컨설턴트가 방향 잡고, AI가 확인/보강 |
| **P6** | **역할별 뷰는 이미 분리되어 있다** | Admin=문서편집, Parent=리포트, Student=가이드앱 |
| **P7** | **빈 상태가 기본이므로 진행률 표시가 핵심** | 빈 그래프가 아니라 진행률 대시보드 |
| **P8** | **같은 시스템 내 주제 중복 체크 필수** | 입학사정관 리스크 방지 — 시스템이 해야 하는 일 |

### 수정된 설계 방향

```
원래 설계:  현재 26섹션 → 100노드 그래프 (전면 교체)
수정 설계:  현재 26섹션 유지 + 점진 개선 (4단계)

핵심 변경:
- 그래프 = 편집 UI가 아님 → 사이드 패널의 "연결 탐색" 모드
- 4-레이어 = 모든 노드에 강제 아님 → 세특/창체에만 탭 추가
- 초기 설계 파이프라인 = AI 자동 아님 → 컨설턴트 보조 도구
- 학부모 뷰 = 그래프 아님 → 월간 리포트 + 진행률
```

## 3. 수정된 로드맵

### Phase G0: 즉시 개선 (1주)

**목표**: 현재 시스템의 스크롤 문제 해결 + 진행률 가시화. 아키텍처 변경 없음.

| # | 작업 | 공수 | 효과 |
|---|------|------|------|
| G0-1 | **빈 섹션 자동 접기** — 데이터 없는 DocSection을 1줄로 축소 ("수상경력 — 해당 없음 [+]") | 0.5일 | 페이지 높이 40-60% 감소 |
| G0-2 | **진행률 대시보드** — 사이드바 최상단에 "기록 7/9 · 진단 미완 · 설계 2/7 · 전략 0/6" 배지 | 0.5일 | 전체 현황 한눈에 파악 |
| G0-3 | **에러 부분 처리** — 쿼리 1개 실패 시 해당 섹션만 에러, 나머지 정상 렌더 | 0.3일 | 페이지 사망 방지 |
| G0-4 | **useAutoSave 보강** — beforeunload + visibilitychange flush + 지수 백오프 3회 | 0.2일 | 데이터 손실 방지 |

**상태**: ✅ 완료 (2026-03-21) — 빌드 성공 + 143 테스트 통과
**검증**: 컨설턴트 피드백 대기

| G1-2 | **📘 가이드 탭 연결** — ExplorationGuidePanel의 해당 과목 배정 가이드를 탭 내부에 표시 | 0.5일 | 기존 기능 재배치 (신규 코드 아님) |

### Phase G2: 연결 탐색 + 초기 설계 보조 (2주)

**목표**: 노드 간 관계를 인라인 크로스레퍼런스로 표면화. 초기 설계 워크플로 추가.

|---|------|------|------|

| G2-2 | **사이드 패널 "연결" 앱** — 현재 선택된 섹션의 관련 노드 목록 (1-hop) | 1일 | on-demand 관계 탐색 |
| G2-3 | **학생 온보딩 위저드** — target_major + school_name 수집 | 1일 | 초기 설계 전제 조건 |
| G2-4 | **교과선택 추천 뷰** — target_major 기반 MAJOR_RECOMMENDED_COURSES 시각화 | 0.5일 | 교과 이수 가이드 |

| G2-6 | **같은 학교 주제 중복 체크** — 가이드 배정/세특 방향 설정 시 동일 학교 학생 주제 겹침 경고 | 1일 | 입학사정관 리스크 방지 |
| G2-7 | **학부모 월간 리포트** — 기존 weekly/monthly 리포트에 생기부 진행률 섹션 추가 | 1일 | 학부모 안심 |

## 4. 원래 설계(v1.0) vs 수정 설계(v1.1) 비교

| 항목 | v1.0 (원래) | v1.1 (수정) |
|------|-----------|-----------|
| **UI 패러다임** | 100노드 그래프 카드 뷰 | 현재 문서 뷰 유지 + 섹션별 레이어 탭 |
| **레이어** | 모든 노드에 4레이어 강제 | 세특=4탭, 창체=4탭, 행특=3탭, 독서=2탭 (유연) |
| **엣지 시각화** | 노드 간 선 연결 | 인라인 크로스레퍼런스 칩 + 사이드 패널 |
| **초기 설계** | AI 자동 파이프라인 | 컨설턴트 보조 (추천 → 확인 → 배정) |
| **학부모 뷰** | 그래프 요약 | 월간 리포트 + 진행률 |
| **학생 뷰** | 노드 카드 | 투두 리스트 (기존 /guides 유지) |
| **쿼리 아키텍처** | 노드별 재설계 | 현재 유지 + select 필터 |
| **구현 기간** | 11~19주 | **~6주** (G0~G3) |
| **리스크** | 전면 리라이트 — 롤백 불가 | 점진 개선 — 각 단계 독립 배포 가능 |
