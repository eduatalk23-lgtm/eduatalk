## 1. 서비스 개요

### 1.1 서비스 명
**TimeLevelUp (EduaTalk)**

### 1.2 서비스 정의
학원(학습 관리 기관)을 위한 AI 기반 통합 학습 관리 플랫폼

### 1.3 핵심 가치 제안
- **AI 기반 맞춤형 학습 플랜**: 학생 성적 분석을 통한 개인화된 학습 경로 자동 생성
- **자동 스케줄링**: 복잡한 시간 제약 조건을 고려한 최적 학습 스케줄 자동 생성
- **실시간 모니터링**: 학습 진행 상황 실시간 추적 및 피드백
- **멀티테넌트 SaaS**: 여러 프렌차이즈 지점이 독립적으로 사용 가능

### 1.4 대상 사용자
- **학원/학습관**: 학생 관리 및 커리큘럼 운영
- **학생**: 개인 학습 플랜 수립 및 실행
- **학부모**: 자녀 학습 현황 모니터링
- **관리자/컨설턴트**: 학생 관리 및 플랜 검토

## 3. 주요 기능 정의

### 3.1 플랜 생성 및 관리

#### 3.1.1 7단계 플랜 생성 위자드
**비즈니스 로직**:
1. **Step 1: 기본 정보** - 플랜 이름, 블록세트(시간표), 학습 기간, 요일 선택
2. **Step 2: 블록 및 제외일** - 학습 시간대 설정, 학원 스케줄 임포트, 제외일 지정
3. **Step 3: 스케줄 확인** - 주간별 스케줄 미리보기, 타임라인 시각화
4. **Step 4: 콘텐츠 선택** - 마스터/학생 콘텐츠 선택, 슬롯 모드 지원
5. **Step 5: 추천 콘텐츠** - AI 기반 콘텐츠 추천, 과목 균형 차트
6. **Step 6: 최종 검토** - 콘텐츠 할당 확인, 전략/취약 과목 에디터
7. **Step 7: 결과 확인** - 생성된 플랜 목록, 활성화 다이얼로그

**핵심 비즈니스 규칙**:
- 오토세이브: 2초 디바운스 자동 저장
- 모드별 분기: 일반/템플릿/캠프/관리자 모드별 다른 플로우
- 상태 관리: draft → saved → active 상태 전이

#### 3.1.2 AI 기반 플랜 생성
**비즈니스 로직**:
- 학생 성적 분석 (모의고사, 내신, 학교 성적)
- 수준 분석 기반 학습 분량 추천
- 전략과목/취약과목 자동 도출
- 학습 범위 분할 및 소요시간 계산
- 복잡한 시간 제약 조건 고려한 스케줄링

#### 3.1.3 슬롯 모드 (캠프)
**비즈니스 로직**:
- 관리자가 슬롯 템플릿 구성
- 학생이 슬롯에 콘텐츠 연결
- 연결 슬롯/배타 슬롯 관계 처리
- AI 기반 슬롯 추천

### 3.2 학습 실행 및 추적

#### 3.2.1 컨테이너 기반 플랜 관리 (Today)
**비즈니스 로직**:
- **Unfinished (미완료)**: 어제 완료하지 못한 플랜들, 우선순위 1
- **Daily (일간)**: 오늘 해야 할 플랜들, 우선순위 2
- **Weekly (주간)**: 이번 주 내 해야 할 플랜들, 우선순위 3
- 컨테이너 간 플랜 이동 (드래그앤드롭)
- Ad-hoc 플랜 추가

#### 3.2.2 타이머 시스템
**비즈니스 로직**:
- 상태 머신: NOT_STARTED → RUNNING → PAUSED → COMPLETED
- 오프라인 큐 시스템 (네트워크 끊김 시 로컬 저장)
- 멀티 디바이스 세션 충돌 감지
- 일시정지 시간 별도 추적

#### 3.2.3 이월(Carryover) 시스템
**비즈니스 로직**:
- 완료되지 않은 플랜 → 다음 학습일로 자동 이월
- 이월 횟수 추적
- 관리자 대량 이월 처리 기능

### 3.3 성적 관리

#### 3.3.1 3종 성적 통합 관리
**비즈니스 로직**:
- **모의고사 (MockScore)**: 전국 모의고사 점수, 백분위, 등급, 원점수
- **학교 성적 (SchoolScore)**: 중간/기말고사 성적, 학교별 석차, 등급
- **내신 (InternalScore)**: 내신 등급, 과목별 세부 사항
- 과목별 추이 분석
- 평균 등급 자동 계산
- 취약 과목 자동 도출

### 3.4 출석 관리

#### 3.4.1 출석 체크 시스템
**비즈니스 로직**:
- QR코드 스캔 출석 체크
- 수동 출석 기록
- 출석 상태: 출석/지각/조퇴/결석/기타
- SMS 자동 발송 연동 (학부모에게 출석 알림)
- 출석 통계 자동 계산
- 출석률 기반 위험 지수 분석

### 3.5 캠프 운영

#### 3.5.1 캠프 템플릿 및 초대 시스템
**비즈니스 로직**:
- 캠프 템플릿 생성 (윈터/썸머/파이널 캠프)
- 슬롯 템플릿 구성
- 학생 초대 발송 및 상태 관리 (pending → accepted/declined)
- 캠프 플랜 자동 생성
- 캠프 라이브 현황 모니터링
- 캠프 알림 및 리마인더

### 3.6 분석 및 리포트

#### 3.6.1 학습 분석
**비즈니스 로직**:
- 주간 메트릭 수집 (학습 시간, 완료률, 목표 달성률)
- 취약 과목 분석
- 위험 지수 계산 (Risk Engine)
- 추천 엔진 결과
- 연속성 지표, 집중도 점수
- 피로도 점수 및 적응형 스케줄링

#### 3.6.2 코칭 리포트
**비즈니스 로직**:
- 주간/월간 학습 리포트 생성
- 성적 추이 분석
- 학습 패턴 분석
- 개선 제안 제공

## 5. 기술 요구사항

### 5.1 기술 스택

#### 프론트엔드
- **Next.js 16.0.10**: App Router 기반 프레임워크
- **React 19.2.0**: UI 라이브러리
- **TypeScript 5**: 타입 안전성
- **Tailwind CSS 4**: 유틸리티 CSS 프레임워크
- **React Query 5.90.10**: 서버 상태 관리
- **Zustand 5.0.9**: 클라이언트 상태 관리

#### 백엔드
- **Next.js Server Actions**: 서버 로직
- **Next.js API Routes**: REST API
- **Supabase**: BaaS (PostgreSQL, Auth, Realtime, Storage)

#### 인프라
- **Vercel**: 호스팅
- **Supabase Cloud**: 데이터베이스 호스팅

### 5.2 아키텍처 요구사항

#### 멀티테넌트 구조
- **방식**: 테넌트 ID 기반 격리 (Shared Database)
- 모든 테이블에 `tenant_id` 컬럼
- Row Level Security (RLS) 정책으로 데이터 격리

#### 데이터베이스 설계
- **핵심 테이블**: tenants, users, students, plan_groups, student_plans
- **관계**: 테넌트 → 학생 → 플랜 그룹 → 개별 플랜
- **인덱싱**: 성능 최적화를 위한 전략적 인덱스

#### 보안 요구사항
- HTTPS 통신 필수
- Row Level Security (RLS) 정책
- 역할 기반 접근 제어 (RBAC)
- API Rate Limiting

#### 성능 요구사항
- 평균 응답 시간: 2초 이하
- 동시 사용자: 최소 1,000명 지원
- 데이터베이스 쿼리 최적화
- React Query 캐싱 전략

## 6. 사용자 시나리오

### 6.1 학생 온보딩 시나리오

1. **회원가입** → 이메일/비밀번호 또는 소셜 로그인
2. **학생 설정** (`/student-setup`) → 기본 정보 입력 (이름, 학교, 학년), 테넌트 선택
3. **콘텐츠 등록** (`/contents`) → 마스터 콘텐츠 선택 또는 커스텀 콘텐츠 등록
4. **블록세트 설정** (`/blocks`) → 기본 시간표 선택, 학원 일정 등록
5. **첫 플랜 생성** (`/plan/new-group`) → 7단계 위자드 진행
6. **학습 시작** (`/today`) → 오늘의 플랜 실행

### 6.2 일일 학습 시나리오

1. **오전**: 출석 체크 (QR 스캔) → 오늘의 플랜 확인 → 미완료 플랜 확인
2. **학습 중**: 플랜 선택 → 타이머 시작 → 학습 진행 → 일시정지 (필요시) → 학습 완료 → 진도 입력 → 다음 플랜으로 이동
3. **저녁**: 하루 학습 마무리 → 미완료 플랜 이월 처리 → 출석 종료 체크

### 6.3 관리자 운영 시나리오

#### 일간 업무
- 출석 현황 확인 (`/admin/attendance`)
- 학습 진행률 모니터링 (`/admin/dashboard`)
- 이상 징후 학생 확인 (위험 지수)

#### 주간 업무
- 플랜 이월 처리 (`/admin/students/[id]/plans`)
- 상담 진행 및 기록 (`/admin/consulting`)
- SMS 발송 (`/admin/sms`)

#### 정기 업무
- 학생 관리 (등록/수정/반 배정)
- 콘텐츠 관리 (마스터 교재/강의 등록)
- 캠프 운영 (템플릿 생성, 초대 발송)
- 성적 입력 및 분석

### 6.4 캠프 운영 시나리오

#### 준비 단계 (관리자)
1. 캠프 템플릿 생성 (`/admin/camp-templates`) → 기본 정보, 슬롯 템플릿 구성, 리마인더 설정
2. 학생 초대 발송 → 초대 상태: pending → accepted/declined

#### 진행 단계 (학생)
3. 초대 수락
4. 슬롯에 콘텐츠 연결 (Step 4까지)
5. 캠프 플랜 자동 생성
6. 캠프 학습 진행 (`/camp/today`)

#### 모니터링 (관리자)
7. 캠프 라이브 현황 확인 (`/admin/camp-live`)
8. 캠프 알림 발송 (`/admin/camp-alerts`)
9. 진행률 모니터링

## 부록: 핵심 비즈니스 규칙

### 플랜 상태 전이
- `draft` → `saved` → `active`
- `active` → `paused` → `completed`
- 미완료 플랜 → 자동 이월

### 역할별 권한
- **학생**: 자신의 플랜 생성/수정/실행
- **관리자**: 모든 학생 플랜 관리, 대리 생성
- **학부모**: 조회 전용
- **슈퍼관리자**: 시스템 전체 관리

### 멀티테넌트 격리
- 모든 데이터는 `tenant_id`로 격리
- RLS 정책으로 자동 필터링
- 테넌트 간 데이터 접근 불가
