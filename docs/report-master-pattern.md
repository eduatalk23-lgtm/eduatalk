# Report Master Pattern (학생 리포트 마스터 표시 패턴)

> **목적**: TimeLevelUp 학생 리포트의 모든 정량 섹션이 따르는 시각 계약(visual contract)
> **배경**: 바이브온의 "스냅샷 + 코호트 비교" 패턴을 모방하지 않고, 우리 강점인 "시계열 + 활동 인용 + 다음 액션"을 통일된 카드로 표현

## 1. 마스터 패턴 (정량 섹션 공통)

### 예시: 학업역량 카드

```
┌────────────────────────────────────────────────┐
│ 학업역량                          [성장 중]    │  ← ① 헤더
├────────────────────────────────────────────────┤
│  현재     │  추이              │  목표갭       │  ← ② 핵심 지표 3종
│  78점     │  ↗ +5점            │  -7점         │
│           │  (vs 2개월 전)     │  (목표 85점)  │
├────────────────────────────────────────────────┤
│  100 ┤                                ●        │  ← ③ 시각화
│   85 ┤  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ (목표)    │     (시계열 또는 진척 막대)
│   78 ┤                          ●              │
│   73 ┤              ●                          │
│      └─────────────────────────────────         │
│        12월   2월   4월        목표             │
├────────────────────────────────────────────────┤
│ 점수 변화의 근거 활동                          │  ← ④ 근거 활동
│ · "환경호르몬 미꾸라지 실험" (2-1, 화학)        │     (LLM이 인용)
│ · "세포 호흡 프로젝트 4차시" (3-1, 생명과학)   │
│ · "엔트로피 자율 발표" (3-1, 수학)              │
├────────────────────────────────────────────────┤
│ AI 코멘트                                      │  ← ⑤ 변화 해석 + 다음 액션
│ 지난 2개월간 5점 상승. 특히 화학·생명과학 탐구 │
│ 활동에서 학업태도가 개선되었음. 다음 단계:     │
│ 화학 II 진로선택을 활용해 심화 탐구를 1건 추가 │
│ 하면 목표선 도달 가능.                         │
└────────────────────────────────────────────────┘
```

## 2. 5개 슬롯 정의

| 슬롯 | 역할 | 필수 | 비고 |
|---|---|---|---|
| ① 헤더 | 항목명 + 한 단어 평가 라벨 | ✅ | 라벨 톤: positive / neutral / caution |
| ② 핵심 지표 3종 | 현재 / 추이 / 목표갭 | ✅ | 슬롯 비울 수 없음. 데이터 부족 시 "—" 표시 |
| ③ 시각화 | 시계열 차트 또는 진척 막대 | ✅ | 데이터 1개뿐이면 진척 막대로 fallback |
| ④ 근거 활동 | LLM이 인용한 학생의 실제 활동 (2~3개) | ✅ | **빈 채로 노출 금지** |
| ⑤ AI 코멘트 | 변화 해석 + 다음 액션 (한 단락) | ✅ | "다음 액션"이 없으면 마스터 패턴 위반 |

> ⚠ ④와 ⑤는 우리 강점의 핵심. 둘 중 하나라도 빠지면 바이브온의 스냅샷 패턴으로 회귀함.

## 3. 데이터 소스 매핑

| 카드 슬롯 | 데이터 소스 | 위치 | 신규 작업 |
|---|---|---|---|
| ① 헤더 라벨 | LLM 분류 (4단계: 우수/성장/안정/주의) | `phase-s3-diagnosis.ts` | 라벨링 함수 |
| ② 현재 점수 | `competency_scores` 최신 row | dual tracking 활용 | — |
| ② 추이 (delta) | `competency_scores` 시점간 diff | dual tracking 활용 | **diff 계산 fetcher 신규** |
| ② 목표갭 | `students.target_school_tier` 기준 임계값 | leveling engine | — |
| ③ 시각화 | `competency_scores` 시계열 (학년/학기 단위) | dual tracking 활용 | **시계열 fetcher 신규** |
| ④ 근거 활동 | `edges` 가중치 상위 노드 → 활동 텍스트 | `phase-s2-edges` | LLM 인용 프롬프트 추가 |
| ⑤ AI 코멘트 | LLM 생성 (변화 + 다음 액션) | 신규 LLM 호출 | 프롬프트 신규 |

> 핵심 인사이트: **모든 데이터는 이미 DB에 있음.** "diff 계산 fetcher"와 "LLM 코멘트 프롬프트" 두 가지만 신규로 만들면 됨.

## 4. 변형 패턴

### 4-1. 활동 섹션 (자율/동아리/진로/세특)

```
┌────────────────────────────────────────────────┐
│ 동아리활동                  [충실히 기록 중]   │
├────────────────────────────────────────────────┤
│  분량        │  개인기록    │  연결도          │
│  84%         │  67%         │  강함 ↑          │
├────────────────────────────────────────────────┤
│ 우수 평가 문장 (학년·역량 태그)                │
│ · "1인 1탐구 주제로 시스템반도체와..."         │
│   [3학년] [학][진]                             │
│ · "과학실험탐구인테그랄2반(31시간) 동아리..."  │
│   [2학년] [학][진][공]                         │
├────────────────────────────────────────────────┤
│ AI: 이 동아리 활동은 화학·생명과학 세특과      │
│ 강하게 연결되어 전공 일관성을 보여줍니다.      │
│ 다음 단계: 발표 영상 자료를 1건 추가하면...    │
└────────────────────────────────────────────────┘
```

② 슬롯이 "분량/개인기록/연결도"로 변경. ③ 시각화는 3중 도넛 또는 막대.

### 4-2. 보완 섹션 (전략/액션)

```
┌────────────────────────────────────────────────┐
│ 학업역량 보완                  [우선순위 1]    │
├────────────────────────────────────────────────┤
│ 약점: 세특_사회 (보완필요)                     │
├────────────────────────────────────────────────┤
│ Why                                            │
│ · 사회 4과목 중 3개에서 탐구정신 보완필요      │
│ · 사회 평균 등급 1.83 (전공 연계 평균 1.56)    │
├────────────────────────────────────────────────┤
│ Action                                         │
│ 다음 학기 "생활과 윤리" 진로선택 + 사회 탐구   │
│ 주제로 "기술 발전과 정보 보호법..." 1건 도전.  │
└────────────────────────────────────────────────┘
```

③ 시각화 생략, ④/⑤가 Why/Action으로 변형.

### 4-3. 마스터 패턴이 적용 안 되는 섹션

| 섹션 | 이유 | 대체 |
|---|---|---|
| determineVerdicts (5단계 배치) | 카테고리형 결과 | 5색 게이지 바 (안정~위험) |
| 워드클라우드 | 시각화 자체가 다름 | 별도 SVG (단, 우선순위 낮음) |
| 추천 도서 | 카탈로그형 | 카드 리스트 |
| 면접 예상질문 | 텍스트 리스트 | Q&A 아코디언 |

## 5. 컴포넌트 인터페이스 (TypeScript)

```typescript
// components/molecules/MasterPatternCard/types.ts

export type EvaluationTone = 'positive' | 'neutral' | 'caution';

export interface MetricSlot {
  current: { value: number; unit: string };
  trend: { delta: number; period: string }; // "+5", "vs 2개월 전"
  targetGap: { value: number; targetLabel: string }; // -7, "목표 85점"
}

export interface VisualizationSlot {
  type: 'timeseries' | 'progress' | 'triple-donut';
  data: Array<{ label: string; value: number }>;
  targetLine?: number;
}

export interface EvidenceItem {
  text: string;
  grade: number;
  semester?: number;
  category: 'setek' | 'changche' | 'reading';
  competencyTags?: Array<'학' | '진' | '공'>;
}

export interface AiCommentSlot {
  interpretation: string; // 변화 해석
  nextAction: string;     // 다음 액션 (필수)
}

export interface MasterPatternCardProps {
  // ① 헤더
  title: string;
  evaluation: { label: string; tone: EvaluationTone };

  // ② 핵심 지표
  metrics: MetricSlot;

  // ③ 시각화
  visualization: VisualizationSlot;

  // ④ 근거 활동 (2~3개)
  evidence: EvidenceItem[];

  // ⑤ AI 코멘트
  aiComment: AiCommentSlot;
}
```

### 검증 규칙 (런타임)
- `evidence.length >= 1` (빈 배열 금지)
- `aiComment.nextAction` 필수 (빈 문자열 금지)
- `metrics.trend` 데이터 1개뿐이면 "—" 표시 (거짓 추세선 금지)

## 6. 비교 기준 정책 (정직한 코호트)

| 비교 대상 | 사용 가능? | 라벨 표기 |
|---|---|---|
| 과거의 나 (시점간 diff) | ✅ 즉시 가능 | "지난 진단 대비" |
| 목표 권장선 (우리 임계값) | ✅ 가능 | **"권장 기준" 명시** (모델 아님) |
| 우리 동급생 코호트 | 🟡 N≥30일 때만 | "유사 학생 N명 평균" |
| 전국 합격자 코호트 | ❌ 데이터 없음 | **흉내 금지** |

> 가장 중요: "지원적정구간 평균" 같은 표기는 우리 데이터로 못 만듦. 만들면 거짓 정밀도가 됨.

## 7. Anti-patterns (하지 말 것)

| ❌ 안 됨 | 이유 |
|---|---|
| "전국 합격자 평균 대비 상위 X%" | 데이터 없음, 거짓 정밀도 |
| AI 코멘트가 통계만 해석 ("점수가 높습니다") | 우리 강점인 활동 인용 누락 |
| ② 슬롯에 "현재"만 있고 "추이"가 없음 | 스냅샷 패턴 회귀 |
| ④ 근거 활동이 빈 채로 노출 | 마스터 패턴 위반 |
| ⑤ "다음 액션"이 일반론 ("탐구활동을 더 하세요") | 컨설팅 가치 없음 |
| ① 헤더 라벨이 부정적 ("부족", "미달") | 외부 메시지 톤과 충돌 — caution까지만 |

## 8. 적용 대상 (Phase 1)

### 마스터 패턴으로 교체할 컴포넌트
- `DiagnosisSection` 내부 역량 카드 3종 (학업/진로/공동체)
- `TimeSeriesSection` → 마스터 패턴 카드에 임베드
- `ContentQualitySection` (5축 품질) → 마스터 패턴 5번 반복
- `GrowthTrajectorySection` → 마스터 패턴 변형

### 마스터 패턴으로 교체 안 함
- `PlacementVerdictSection` (Phase 1.1, 신규) — 5색 게이지
- `ExecutiveSummarySection` (Phase 1.2, 신규) — 텍스트 위주
- `WordCloudSection` — Phase 3 이후
- `InterviewSection` — Q&A 형식 유지
- `BypassMajorSection` — 표 형식 유지

## 10. 한 줄 정리

> **현재(Where) + 추이(How) + 갭(Whither) + 근거(Why) + 액션(What next)** — 다섯 슬롯이 모두 채워지지 않으면 우리 카드가 아니다.
