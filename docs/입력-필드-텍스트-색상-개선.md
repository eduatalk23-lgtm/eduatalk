# 입력 필드 텍스트 색상 개선

**작업 일자**: 2024년 12월  
**작업 범위**: 입력 필드 가독성 개선

## 문제점

입력 필드의 텍스트와 placeholder 색상이 너무 옅어서 사용자가 입력한 내용이나 예시 텍스트를 잘 보기 어려웠습니다.

## 해결 방법

입력 필드의 텍스트 색상과 placeholder 색상을 더 진하게 변경하여 가독성을 향상시켰습니다.

### 변경된 색상

- **입력 텍스트**: `text-gray-900` (매우 진한 회색)
- **placeholder**: `placeholder:text-gray-600` (진한 회색, 기존 `text-gray-400`에서 변경)

## 수정된 파일

### 1. 계정 관리 페이지 (`app/(student)/settings/account/page.tsx`)

3개의 비밀번호 입력 필드에 텍스트 색상과 placeholder 색상을 추가했습니다.

```typescript
// 변경 전
className="rounded-lg border border-gray-300 px-3 py-2 focus:border-indigo-500..."

// 변경 후
className="rounded-lg border border-gray-300 px-3 py-2 text-gray-900 placeholder:text-gray-600 focus:border-indigo-500..."
```

### 2. 공통 Input 컴포넌트 (`components/atoms/Input.tsx`)

프로젝트 전체에서 사용되는 Input 컴포넌트의 기본 스타일을 변경했습니다.

- `placeholder:text-gray-400` → `placeholder:text-gray-600`
- `text-gray-900` 명시적으로 추가

### 3. FormInput 컴포넌트 (`components/ui/FormInput.tsx`)

FormInput 컴포넌트에도 동일한 색상 스타일을 추가하여 일관성을 유지했습니다.

- `text-gray-900` 추가
- `placeholder:text-gray-600` 추가

## 영향 범위

이 변경사항은 다음과 같은 효과를 가집니다:

1. **계정 관리 페이지**: 비밀번호 입력 필드의 가독성 향상
2. **전체 프로젝트**: `Input` 컴포넌트를 사용하는 모든 페이지에 자동 적용
3. **일관성**: 모든 입력 필드에서 동일한 색상 규칙 적용

## 결과

- ✅ 입력 필드의 텍스트가 더 진하고 명확하게 보임
- ✅ placeholder 텍스트도 더 잘 보여 사용자가 예시를 쉽게 파악 가능
- ✅ 전체 프로젝트에서 일관된 입력 필드 스타일 유지
- ✅ Linter 에러 없음
