## 1. 개요

### 1.1 배경

TimeLevelUp의 생기부 시스템은 Phase 1~9.4 + CMS C1~C5 + Agent A~E + 우회학과 고도화까지 전체 구현이 완료된 상태이다.
현재 시스템은 **4단계 파이프라인(기록→진단→설계→전략)**을 단일 스크롤 페이지(26섹션, ~7,800px)로 제공하며, 기능적으로는 완성되었으나 구조적으로 다음 한계가 있다:

1. **정보 분절**: 같은 과목의 가이드/가안/실기록/분석이 서로 다른 스테이지에 분산
2. **관계 비가시성**: 세특↔창체↔독서↔행특 간 연결이 시스템적으로 존재하나 UI에서 보이지 않음
3. **빈 상태 대응 부재**: 신규 학생(데이터 0건)에 대한 초기 설계 워크플로가 없음
4. **역할별 뷰 부재**: 컨설턴트/학부모/학생이 같은 페이지를 같은 방식으로 봄

### 1.2 목표

생기부 문서를 **공간적 좌표계(spatial anchor)**로 활용하여, 각 영역에 **4개 깊이 레이어(가이드→가안→실기록→분석)**를 부여하고, 영역 간 **관계(엣지)**를 시각화하는 **그래프 기반 컨설팅 시스템**을 구축한다.

### 1.3 핵심 가치

| 기존 서비스 (진학사/바이브온) | TimeLevelUp 그래프 시스템 |
|---|---|
| 생기부 → 분석 → 점수 (일방향) | 생기부 = 관계를 가진 살아있는 그래프 |
| 분석과 기록이 분리 | 한 영역에서 가이드→가안→실기록→분석 통합 |
| 빈 상태 = 분석 불가 | 빈 상태 = 설계의 시작점 (AI가 계획 먼저 생성) |
| 입력 폼과 결과 리포트 분리 | 입력 폼이 곧 분석 뷰 |

### 1.4 참조 서비스 패턴

| 서비스 | 차용 패턴 | 적용 |
|--------|----------|------|
| **Turnitin Feedback Studio** | 동일 문서 위 토글 가능한 레이어 (원본/유사도/피드백/루브릭) | 세특 위에 가이드/가안/분석 레이어 |
| **Revit View Discipline** | 동일 건물 위 분야별 뷰 (건축/구조/MEP/Coordination) | 생기부 위 역할별 렌즈 |
| **Crowdin/Phrase TMS** | 동일 세그먼트의 워크플로 단계 (Source→Translation→Review→Approved) | 가이드→가안→실기록→분석 워크플로 |
| **Epic EMR Chart Review** | 환자 차트를 공간 앵커로, 탭별 다른 데이터 레이어 | 학생 생기부를 공간 앵커로, 레이어별 다른 깊이 |

## 2. 그래프 구조 정의

### 2.1 노드 (Nodes)

생기부의 모든 주요 영역이 노드이며, 각 노드는 **학년별로 인스턴스화**된다.

#### 노드 유형

| 노드 유형 | 인스턴스 단위 | 예시 | 레이어 적용 |
|----------|-------------|------|-----------|
| **세특** | 학년 × 과목 | "2학년 수학 세특" | ✅ 4레이어 전체 |
| **개인세특** | 학년 × 제목 | "2학년 융합탐구" | ✅ 4레이어 전체 |
| **창체(자율)** | 학년 | "2학년 자율활동" | ✅ 4레이어 전체 |
| **창체(동아리)** | 학년 | "2학년 동아리활동" | ✅ 4레이어 전체 |
| **창체(진로)** | 학년 | "2학년 진로활동" | ✅ 4레이어 전체 |
| **행특** | 학년 | "2학년 행동특성" | 📝📄🔍 (가이드 해당 없음) |
| **독서** | 학년 × 도서 | "1학년 코스모스" | 📄🔍 (실기록+분석) |
| **교과성적** | 학년 × 과목 | "2학년 수학 성적" | 📄🔍 (실기록+분석) |
| **교과선택** | 학년 | "2학년 이수 과목 목록" | 📘📄 (추천+실이수) |
| **출결** | 학년 | "2학년 출결" | 📄 (실기록만) |

#### 노드 수 추정 (3학년 학생 기준)

```
세특: 3학년 × ~12과목 = ~36개
개인세특: 3학년 × ~2개 = ~6개
창체: 3학년 × 3영역 = 9개
행특: 3개
독서: ~10개
교과성적: 3학년 × ~12과목 = ~36개
교과선택: 3개
출결: 3개
───────────────────
총: ~100+ 노드 (단, UI에서는 학년별 그룹핑으로 압축)
```

### 2.2 레이어 (Layers / Depth)

각 노드는 최대 4개의 깊이 레이어를 가진다. 레이어는 **컨설팅 워크플로의 단계**에 대응한다.

#### 레이어 정의

| 레이어 | 아이콘 | 의미 | 데이터 소스 | 생성 주체 |
|--------|--------|------|-----------|----------|
| **📘 가이드** | ● (1단계) | "여기서 이런 탐구를 하면 좋겠다" | `exploration_guide_assignments`, `MAJOR_RECOMMENDED_COURSES` | AI 추천 → 컨설턴트 확정 |
| **📝 가안** | ● (2단계) | "이런 방향으로 활동/세특을 만들자" | `SetekGuideItem`, `activity_summaries`, 메모, 채팅 | AI 생성 → 컨설턴트 수정 |
| **📄 실기록** | ● (3단계) | "실제로 이렇게 기록되었다" | `student_record_seteks`, `changche`, `haengteuk`, `reading` | 교사 작성 (Import/수동입력) |
| **🔍 분석** | ● (4단계) | "이 기록에서 이런 역량이 드러난다" | `activity_tags`, `competency_scores`, 하이라이트 | AI 자동 |

#### 레이어별 기존 구현 매핑

| 레이어 | 기존 기능 | 기존 위치 (스테이지) | 신규 위치 (노드 내부) |
|--------|----------|-------------------|-------------------|
| 📘 가이드 | CMS 탐구 가이드 배정 | 설계 > 활동 가이드 | 해당 과목 노드 📘탭 |
| 📝 가안 | 세특 방향 가이드 (AI) | 설계 > 세특 방향 가이드 | 해당 과목 노드 📝탭 |
| 📝 가안 | 활동 요약서 (AI) | 설계 > 활동 요약서 | 해당 영역 노드 📝탭 |
| 📝 가안 | 컨설턴트 메모 | 사이드 패널 > 메모 | 해당 영역 노드 📝탭 내 메모 |
| 📝 가안 | 채팅 논의 | 사이드 패널 > 채팅 | 해당 영역 노드 📝탭 내 대화 |
| 📄 실기록 | 세특 에디터 (자동저장) | 기록 > sec-7 | 해당 과목 노드 📄탭 |
| 📄 실기록 | 창체 에디터 | 기록 > sec-6 | 해당 영역 노드 📄탭 |
| 📄 실기록 | 행특 에디터 | 기록 > sec-9 | 행특 노드 📄탭 |
| 🔍 분석 | AI 하이라이트 | 진단 > 역량분석 | 해당 노드 🔍탭 인라인 |
| 🔍 분석 | 역량 태그 | 진단 > 역량분석 | 해당 노드 🔍탭 태그 |
| 🔍 분석 | 가이드 키워드 반영률 | (미구현) | 해당 노드 🔍탭 신규 |

#### 상태 진행 인디케이터

```
○○○○  시작 전 — 온보딩 필요
●○○○  가이드 배정됨
●●○○  가안(방향) 설정됨
●●●○  실기록 작성됨
●●●●  분석 완료
```

### 2.3 엣지 (Edges / Relations)

노드 간 관계는 입학사정관 평가 관점에서 도출되었다.

#### 엣지 유형 7종

| # | 엣지 유형 | 의미 | 평가 중요도 | 자동 감지 |
|---|----------|------|-----------|----------|
| 1 | **CONTENT_REFERENCE** | 한 영역의 내용이 다른 영역을 직접 언급 | ★★★★★ | △ NLP/키워드 매칭 |
| 2 | **TEMPORAL_GROWTH** | 같은 주제가 학년을 넘어 심화 | ★★★★★ | △ 임베딩 유사도 |
| 3 | **THEME_CONVERGENCE** | 서로 다른 영역이 같은 테마/키워드로 수렴 | ★★★★★ | ○ 키워드 공유 감지 |
| 4 | **COURSE_SUPPORTS** | 교과 선택이 진로 방향을 뒷받침 | ★★★★★ | ✅ `MAJOR_RECOMMENDED_COURSES` |
| 5 | **COMPETENCY_SHARED** | 동일 역량이 여러 영역에서 발현 | ★★★★☆ | ✅ `activity_tags` 조인 |
| 6 | **TEACHER_VALIDATION** | 행특이 다른 영역의 내용을 입증 | ★★★★☆ | △ 텍스트 유사도 |
| 7 | **READING_ENRICHES** | 독서가 세특/창체 탐구를 보강 | ★★★☆☆ | △ `reading_links` + 키워드 |

#### 기존 DB에서의 엣지 매핑

| 엣지 유형 | DB 테이블/메커니즘 | 현재 건수 | 자동화 수준 |
|----------|-------------------|----------|-----------|
| CONTENT_REFERENCE | `storyline_links` | 4건 | AI 자동 (detectInquiryLinks) |
| TEMPORAL_GROWTH | `storyline_links` (학년간) | 포함 | AI 자동 |
| THEME_CONVERGENCE | (미구현) | 0건 | 키워드 공유 감지 필요 |
| COURSE_SUPPORTS | `course-adequacy.ts` (계산) | 런타임 | ✅ 완전 자동 |
| COMPETENCY_SHARED | `activity_tags` (같은 역량 코드) | 120건 | ✅ 완전 자동 |
| TEACHER_VALIDATION | (미구현) | 0건 | NLP 필요 |
| READING_ENRICHES | `reading_links` | 0건 | 수동 (UI 있음) |

## 3. 컨셉 C: 상태 적응형 깊이 뷰

### 3.1 핵심 원리

사용자가 레이어를 선택하지 않는다. **노드가 현재 상태에 따라 최적의 모습을 자동으로 보여준다.**

- 비어있으면 → 📘 가이드가 메인 (무엇을 해야 하는지)
- 방향 설정되면 → 📝 가안이 메인 (어떻게 해야 하는지)
- 기록되면 → 📄 실기록이 메인 (무엇이 기록되었는지)
- 분석되면 → 📄+🔍 통합 뷰 (기록 위에 분석 오버레이)
- [펼치기] → 전체 4레이어 깊이 스크롤 (학부모 프레젠테이션용)

### 3.2 상태별 기본 뷰

## 4. 초기 설계 파이프라인 (Day 1)

### 4.1 문제

데이터 밀도 조사 결과 (2026-03-21 프로덕션 DB):
- 학생 8명 중 **데이터 있는 학생 2명** (김세린 31세특, 김가영 14세특)
- `target_major`: **8명 전원 NULL**
- `school_offered_subjects`: **0건**
- `exploration_guide_assignments`: **0건**
- `student_record_activity_summaries`: **0건**

**레이어 📘과 📝이 완전히 비어있다.** 시스템의 핵심 가치인 "설계→실행→분석" 워크플로가 작동하지 않는다.

### 4.2 학생 온보딩 흐름

초기 설계 파이프라인 이전에 **필수 입력**을 수집하는 온보딩 단계가 필요하다.

```
[학생 등록]
    ↓
[온보딩 위저드]
    ├─ Step 1: 기본정보 확인 (이름, 학년, 학교)
    ├─ Step 2: 목표 전공/계열 설정 (target_major)
    ├─ Step 3: 학교 개설 과목 확인 (school_offered_subjects)
    └─ Step 4: 기존 기록 유무 확인
         ├─ 있음 → Import (PDF/HTML) → 갭 분석
         └─ 없음 → 초기 설계 파이프라인 즉시 실행
```

### 4.3 초기 설계 파이프라인

온보딩 완료 후 시스템이 자동 생성하는 콘텐츠:

| 순서 | 액션 | 입력 | 출력 | 기존 구현 |
|------|------|------|------|----------|
| 1 | **교과선택 추천** | 목표 전공 | 이수 추천 과목 목록 | ✅ `MAJOR_RECOMMENDED_COURSES` 22개 계열 |
| 2 | **탐구 가이드 매칭** | 전공 + 과목 | 과목별 추천 가이드 배정 | ✅ 벡터 검색 7,836건 + 배정 시스템 |
| 3 | **세특 방향 생성** | 전공 + 과목 + 가이드 | 과목별 세특 방향/키워드 | ✅ `generateSetekGuide` AI |
| 4 | **창체 활동 제안** | 전공 + 학년 | 자율/동아리/진로 활동 제안 | △ 로드맵 테이블 있음, 자동 생성 없음 |
| 5 | **3년 로드맵 생성** | 전공 + 학년 + 1~4 종합 | 학기별 실행 계획 타임라인 | △ 테이블 있음, 자동 생성 없음 |
| 6 | **독서 추천** | 전공 + 과목 | 추천 도서 목록 | ✗ 미구현 |

파이프라인 실행 결과:
- 📘 가이드 레이어: 과목별 탐구 가이드 배정됨
- 📝 가안 레이어: 과목별 세특 방향 + 활동 계획 생성됨
- 교과선택 추천: 학생에게 추천 과목 제시
- 3년 로드맵: 학기별 마일스톤 설정

**컨설턴트 확정 필수**: AI가 생성한 모든 콘텐츠는 `status: suggested` 상태. 컨설턴트가 검토/수정 후 `confirmed`로 전환해야 학생에게 공유됨.

### 4.4 진입 시점별 분기

| 진입 시점 | 기존 데이터 | 시스템 대응 |
|----------|-----------|-----------|
| **1학년 시작 전** (12~2월) | 없음 | 3년 풀 로드맵 → 📘📝 전체 생성 |
| **2학년 시작** | 1학년 기록 | Import → 🔍 1학년 분석 → 2~3학년 📘📝 생성 |

| **중간 편입** | 부분 기록 | Import → 갭 분석 → 빈 영역 📘📝 보충 |

| | 전공 교과 성취도 | 전공 관련 과목 성적 | 교과성적 |

### 5.2 입학사정관 읽기 순서 → 그래프 탐색 경로

```
진로활동 (방향 파악)
    ↓
창체 전체 (자율→동아리→진로: 주도성 + 깊이)
    ↓
세특 (과목별 탐구: 진로와 연결되는지)
    ↓
교과성적 + 교과선택 (수치적 검증)
    ↓
행특 (교사가 본 전체 그림: 교차 검증)
```

이것은 **그래프 BFS(너비우선탐색)**이다 — 진로를 시작점으로 연결된 노드를 따라가며 **일관성(consistency)**을 확인.

### 5.3 학년별 기대치

| 학년 | 단계 | 세특 깊이 기대 | 키워드 범위 |
|------|------|-------------|-----------|
| 1학년 | 탐색 (Exploration) | "~에 관심을 갖고 조사함" | 넓은 탐색 |
| 2학년 | 심화 (Deepening) | "~을 분석하고 비교하여" | 좁은 심화 |
| 3학년 | 확장/통합 (Integration) | "논문을 조사하고 ~을 제안함" | 전공 밀접 |

### 5.4 대학별 평가 비중 차이

| 대학 | 학업역량 | 진로역량 | 공동체역량 |

| 고려대 학업우수 | 50% | 30% (자기계발) | 20% |

**시스템 반영**: 목표 대학 설정 시 해당 대학의 평가 비중을 가중치로 적용하여 레이어별 우선순위와 경보 기준을 조정.

## 7. 채팅-노드 연결 설계

### 7.1 현재 상태

```
topic 형식: "record:{student_id}:{section_type}:{identifier}"
예시:
  - "record:0e3e:setek:수학"
  - "record:0e3e:changche:동아리"
  - "record:0e3e:haengteuk:2학년"
```

노드 📝 가안 레이어에서 해당 토픽의 채팅이 인라인으로 표시됨.
