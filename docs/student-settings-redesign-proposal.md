# 학생 마이페이지 개편 제안

## 개편 목표

1. **일관성 있는 UI/UX**: 모든 설정 페이지의 통일된 디자인과 레이아웃
2. **직관적인 네비게이션**: 설정 항목 간 쉬운 이동과 명확한 구조
3. **사용자 경험 개선**: 더 나은 정보 구조와 사용자 플로우
4. **접근성 향상**: 명확한 라벨링과 안내 메시지

## 1. 전체 구조 개편

### 1.1 메인 마이페이지 재구성

#### 현재 문제점
- "마이페이지"라는 이름이 프로필 정보만을 의미하는지 불명확
- 탭 구조가 3개로 나뉘어 있어 정보의 연관성이 떨어짐
- 초기 설정 모드와 일반 모드의 UI 차이가 큼

#### 개편안

**옵션 A: 통합 프로필 페이지 (권장)**
- 페이지명: "프로필" 또는 "내 정보"
- 단일 페이지에 모든 프로필 정보를 섹션으로 구성
- 스크롤 가능한 긴 폼 형태
- 섹션 구분: 기본 정보, 연락처, 입시 정보, 진로 정보

**옵션 B: 개선된 탭 구조**
- 현재 탭 구조 유지하되 UI 개선
- 탭을 더 명확하게 구분 (아이콘 추가)
- 각 탭의 완성도 표시 (진행률 표시)

**권장: 옵션 A**
- 모바일 친화적
- 모든 정보를 한눈에 파악 가능
- 스크롤로 자연스러운 흐름

### 1.2 설정 페이지 통합 구조

#### 제안 구조

```
/settings (메인 설정 페이지)
├── /profile (프로필 정보) - 기존 /settings
├── /notifications (알림 설정)
├── /security (보안 설정)
│   ├── /devices (로그인 기기 관리)
│   └── /account (계정 관리)
└── /preferences (환경 설정) - 향후 확장
```

#### 네비게이션 개선

**사이드바 구조**
```
설정
├── 프로필
├── 알림 설정
└── 보안
    ├── 로그인 기기 관리
    └── 계정 관리
```

**또는 설정 페이지 내부 네비게이션 추가**
- 각 설정 페이지 상단에 설정 메뉴 탭 추가
- 현재 페이지 하이라이트
- 빠른 이동 가능

## 2. 프로필 페이지 개편

### 2.1 페이지 구조

#### 레이아웃
- **최대 너비**: 2xl (일관성 유지)
- **섹션 구분**: 카드 형태로 각 섹션 구분
- **간격**: 섹션 간 `gap-6`

#### 섹션 구성

**1. 기본 정보 섹션**
- 제목: "기본 정보"
- 항목:
  - 이름 (필수)
  - 학교 (선택)
  - 학년 (필수)
  - 생년월일 (필수)
  - 성별 (선택)

**2. 연락처 정보 섹션**
- 제목: "연락처 정보"
- 항목:
  - 본인 연락처
  - 모 연락처
  - 부 연락처
- 설명: "비상 연락을 위한 연락처 정보입니다"

**3. 입시 정보 섹션**
- 제목: "입시 정보"
- 항목:
  - 입시년도 (자동 계산 옵션)
  - 개정교육과정 (자동 계산 옵션)
- 설명: "입시 정보는 학습 계획 수립에 활용됩니다"

**4. 진로 정보 섹션**
- 제목: "진로 정보"
- 항목:
  - 진학 희망 대학교 (최대 3개, 순위별)
  - 희망 진로 계열
- 설명: "진로 정보는 맞춤형 학습 추천에 활용됩니다"

### 2.2 UI 개선사항

#### 초기 설정 모드
- **위치**: 페이지 상단에 고정 배너
- **디자인**: 더 눈에 띄는 디자인 (그라데이션 배경)
- **내용**: 
  - 환영 메시지
  - 진행 단계 표시 (간단한 프로그레스 바)
  - 다음 단계 안내

#### 폼 필드 개선
- **레이블**: 더 명확한 설명 추가
- **도움말**: 각 필드 하단에 도움말 텍스트 (선택적)
- **에러 메시지**: 더 친절한 에러 메시지
- **성공 피드백**: 저장 성공 시 토스트 알림 (기존 인라인 메시지 대신)

#### 저장 버튼
- **위치**: 페이지 하단 고정 (sticky)
- **디자인**: 더 눈에 띄는 디자인
- **상태 표시**: 변경사항이 있을 때만 활성화, 시각적 피드백

### 2.3 기능 개선

#### 자동 계산
- 자동 계산된 값은 읽기 전용으로 표시
- "수동 입력" 버튼으로 전환 가능
- 자동 계산 로직 설명 추가

#### 변경사항 추적
- 브라우저 기본 confirm 대신 커스텀 다이얼로그 사용
- 변경사항 요약 표시

## 3. 알림 설정 페이지 개편

### 3.1 구조 개선

#### 섹션 재구성
- **알림 유형**: 카드 그리드 레이아웃 (2열)
- **알림 시간**: 단일 카드
- **방해 금지 시간**: 단일 카드

#### UI 개선
- 각 알림 유형에 아이콘 추가
- 토글 스위치 디자인 개선
- 설명 텍스트 더 명확하게

### 3.2 기능 개선
- 실시간 저장 (각 토글 변경 시 즉시 저장)
- 또는 변경사항 일괄 저장 (현재 방식 유지)

## 4. 보안 설정 통합

### 4.1 구조 제안

**보안 설정 메인 페이지** (`/settings/security`)
- 보안 관련 설정 개요
- 빠른 액션 카드
  - 로그인 기기 관리 (카드 클릭 → `/settings/security/devices`)
  - 계정 관리 (카드 클릭 → `/settings/security/account`)

**또는 직접 하위 페이지로 이동**
- 사이드바에서 바로 하위 메뉴 접근

### 4.2 로그인 기기 관리 개선

#### UI 개선
- 기기 목록을 테이블 형태로 변경 (더 많은 정보 표시)
- 기기 타입 아이콘 추가 (모바일, 데스크톱 등)
- 마지막 활동 시간 더 명확하게 표시

#### 기능 개선
- 기기 이름 수정 기능
- 기기별 상세 정보 모달

### 4.3 계정 관리 개선

#### 추가 기능 제안
- 이메일 변경 (향후)
- 계정 삭제 (향후)
- 로그인 기록 (향후)

#### UI 개선
- 비밀번호 강도 표시
- 비밀번호 변경 가이드라인 표시

## 5. 네비게이션 개선

### 5.1 사이드바 개선

#### 현재 구조
```
설정
├── 마이페이지
├── 알림 설정
├── 로그인 기기 관리
└── 계정 관리
```

#### 개편안
```
설정
├── 프로필
├── 알림 설정
└── 보안
    ├── 로그인 기기 관리
    └── 계정 관리
```

### 5.2 설정 페이지 내부 네비게이션

각 설정 페이지 상단에 설정 메뉴 추가:

```
[프로필] [알림 설정] [보안]
```

- 현재 페이지 하이라이트
- 클릭 시 해당 페이지로 이동
- 모바일에서는 드롭다운으로 표시

## 6. 디자인 시스템 통일

### 6.1 레이아웃 통일

- **최대 너비**: 모든 설정 페이지 2xl로 통일
- **패딩**: `p-6 md:p-8` 통일
- **간격**: 섹션 간 `gap-6` 통일

### 6.2 컴포넌트 통일

#### 카드 컴포넌트
- 모든 섹션을 카드로 통일
- 일관된 그림자와 보더

#### 버튼 스타일
- Primary: indigo-600
- Secondary: gray-300 border
- Danger: red-600 (로그아웃 등)

#### 메시지 컴포넌트
- 토스트 알림으로 통일 (인라인 메시지 대신)
- 성공/에러/정보 타입 구분

### 6.3 타이포그래피

- 페이지 제목: `text-3xl font-semibold`
- 섹션 제목: `text-xl font-semibold`
- 필드 레이블: `text-sm font-medium`
- 설명 텍스트: `text-sm text-gray-500`

## 10. 상세 설계

### 10.1 프로필 페이지 상세

#### 레이아웃 구조
```
<div className="p-6 md:p-8">
  <div className="mx-auto max-w-2xl">
    <h1>프로필</h1>
    
    {/* 초기 설정 배너 */}
    {isInitialSetup && <InitialSetupBanner />}
    
    <form className="flex flex-col gap-6">
      {/* 기본 정보 섹션 */}
      <SectionCard title="기본 정보">
        <FormField label="이름" required />
        <FormField label="학교" />
        <FormField label="학년" required />
        <FormField label="생년월일" required />
        <FormField label="성별" />
      </SectionCard>
      
      {/* 연락처 정보 섹션 */}
      <SectionCard title="연락처 정보">
        <FormField label="본인 연락처" />
        <FormField label="모 연락처" />
        <FormField label="부 연락처" />
      </SectionCard>
      
      {/* 입시 정보 섹션 */}
      <SectionCard title="입시 정보">
        <FormField label="입시년도" autoCalculate />
        <FormField label="개정교육과정" autoCalculate />
      </SectionCard>
      
      {/* 진로 정보 섹션 */}
      <SectionCard title="진로 정보">
        <FormField label="진학 희망 대학교" />
        <FormField label="희망 진로 계열" />
      </SectionCard>
      
      {/* 저장 버튼 */}
      <StickySaveButton />
    </form>
  </div>
</div>
```

#### 컴포넌트 설계

**SectionCard**
```typescript
type SectionCardProps = {
  title: string;
  description?: string;
  children: ReactNode;
};
```

**FormField**
```typescript
type FormFieldProps = {
  label: string;
  required?: boolean;
  autoCalculate?: boolean;
  helpText?: string;
  error?: string;
  // ... 기타 props
};
```

**StickySaveButton**
```typescript
type StickySaveButtonProps = {
  hasChanges: boolean;
  isSaving: boolean;
  onSubmit: () => void;
  onCancel: () => void;
};
```
