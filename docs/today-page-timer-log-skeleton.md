# 타이머 로그 기능 스켈레톤 UI

## 개요

플랜 타이머의 시작, 일시정지, 재개, 완료 이벤트를 기록하고 표시하는 기능입니다.

## 1. 데이터베이스 구조

### plan_timer_logs 테이블

```sql
CREATE TABLE plan_timer_logs (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  plan_id uuid NOT NULL REFERENCES student_plan(id) ON DELETE CASCADE,
  student_id uuid NOT NULL REFERENCES students(id) ON DELETE CASCADE,
  event_type text NOT NULL CHECK (event_type IN ('start', 'pause', 'resume', 'complete')),
  timestamp timestamptz NOT NULL DEFAULT now(),
  duration_seconds integer, -- 이벤트 발생 시점의 누적 학습 시간 (초)
  note text, -- 선택적 메모
  created_at timestamptz DEFAULT now()
);
```

## 3. 이벤트 타입별 아이콘 및 색상

- **시작 (start)**: 🟢 초록색, Play 아이콘
- **일시정지 (pause)**: ⏸️ 노란색, Pause 아이콘
- **재개 (resume)**: ▶️ 파란색, Play 아이콘
- **완료 (complete)**: ✅ 초록색, CheckCircle 아이콘

## 4. 구현 요소

### 서버 액션
- `recordTimerLog()`: 타이머 이벤트 기록
- `getTimerLogs()`: 플랜의 타이머 로그 조회

### 컴포넌트
- `TimerLogSection`: 타이머 로그 표시 컴포넌트
- TimeCheckSection에 통합

### 데이터 흐름
```
사용자 액션 (시작/일시정지/재개/완료)
  ↓
todayActions (startPlan/pausePlan/resumePlan/completePlan)
  ↓
recordTimerLog() 호출
  ↓
plan_timer_logs 테이블에 저장
  ↓
페이지 새로고침 또는 실시간 업데이트
  ↓
TimerLogSection에서 로그 표시
```

## 5. 표시 정보

각 로그 항목에 포함될 정보:
- 이벤트 타입 (시작/일시정지/재개/완료)
- 발생 시간 (HH:mm:ss)
- 해당 시점의 누적 학습 시간 (선택사항)
- 이벤트 간격 (선택사항)

## 6. 정렬 및 필터링

- 최신순 정렬 (가장 최근 이벤트가 위에)
- 최대 표시 개수 제한 (예: 최근 10개)
- 스크롤 가능한 영역 (로그가 많을 경우)

## 7. 반응형 고려사항

- 모바일: 간결한 리스트 형식
- 데스크톱: 상세한 카드 형식
