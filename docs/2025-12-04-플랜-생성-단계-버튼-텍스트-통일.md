# 플랜 생성 단계 버튼 텍스트 통일

## 작업 일자
2025-12-04

## 문제점
플랜 생성 단계 이동 시 버튼 텍스트가 모드별로 다르게 표시되어 일관성이 없었습니다:
- Step 4: "템플릿 저장하기", "참여 제출하기" 등 모드별로 다른 텍스트
- Step 5: "스케줄 미리보기로 이동" 등 상세한 텍스트
- Step 6: "스케줄 결과 보기" 등 상세한 텍스트

## 수정 내용

### 1. 마지막 단계 판단 로직 추가
- **파일**: `app/(student)/plan/new-group/_components/PlanGroupWizard.tsx`
- **변경 사항**:
  - 각 모드별로 마지막 단계를 판단하는 `isLastStep` useMemo 추가
  - 템플릿 모드: Step 4가 마지막
  - 캠프 모드 (학생): Step 4가 마지막
  - 일반 모드: Step 7이 마지막
  - 캠프 모드 (관리자 continue): Step 7이 마지막

### 2. 버튼 텍스트 통일
- **파일**: `app/(student)/plan/new-group/_components/PlanGroupWizard.tsx`
- **변경 사항**:
  - 마지막 단계면 "완료", 아니면 "다음"으로 통일
  - 모든 상세 텍스트 제거 ("템플릿 저장하기", "참여 제출하기", "스케줄 미리보기로 이동", "스케줄 결과 보기" 등)

## 마지막 단계 판단 로직

```typescript
const isLastStep = useMemo(() => {
  if (isTemplateMode) {
    return currentStep === 4; // 템플릿 모드는 Step 4가 마지막
  }
  if (isCampMode && !isAdminContinueMode) {
    return currentStep === 4; // 캠프 모드 (학생)는 Step 4가 마지막
  }
  return currentStep === 7; // 일반 모드와 캠프 모드 (관리자)는 Step 7이 마지막
}, [currentStep, isTemplateMode, isCampMode, isAdminContinueMode]);
```

## 버튼 텍스트 로직

```typescript
{isPending
  ? "저장 중..."
  : isLastStep
  ? "완료"
  : "다음"}
```
