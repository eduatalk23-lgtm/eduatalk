# 필터 UI 가로 크기 변경 문제 해결

## 📋 문제 상황

마스터 콘텐츠 검색 필터 UI에서 드롭다운의 가로 크기가 계속 변경되는 문제가 발생했습니다.

**문제점:**
- 선택된 옵션의 텍스트 길이에 따라 드롭다운 너비가 변경됨
- 로딩 중일 때와 로딩 완료 후의 크기가 달라짐
- 다른 필터를 선택할 때 레이아웃이 흔들리는 현상

## 🔍 원인 분석

1. **고정 너비 부재**: 드롭다운에 고정 너비가 설정되지 않아 선택된 값의 텍스트 길이에 따라 크기가 변경됨
2. **로딩 상태 처리**: 로딩 중일 때와 완료 후의 텍스트 길이가 달라 레이아웃 변경 발생
3. **반응형 미고려**: 작은 화면에서의 레이아웃 고려 부족

## ✅ 해결 방법

### 1. 최소 너비(min-width) 설정
각 드롭다운 필터에 최소 너비를 설정하여 크기 변화를 방지했습니다.

```tsx
// 개정교육과정
<div className="flex flex-col gap-1 min-w-[160px]">
  <select className="w-full ...">
    {/* ... */}
  </select>
</div>

// 교과, 과목
<div className="flex flex-col gap-1 min-w-[140px]">
  <select className="w-full ...">
    {/* ... */}
  </select>
</div>

// 제목 검색
<div className="flex flex-col gap-1 min-w-[200px] flex-1 max-w-[300px]">
  <input className="w-full ..." />
</div>
```

### 2. w-full 적용
각 select/input 요소에 `w-full` 클래스를 추가하여 부모 컨테이너의 너비를 모두 사용하도록 설정했습니다.

### 3. 로딩 상태에서도 동일한 너비 유지
로딩 중일 때도 동일한 최소 너비를 유지하도록 설정하여 레이아웃 변경을 방지했습니다.

### 4. 제목 검색 필드 최적화
제목 검색 input에 `flex-1`과 `max-w-[300px]`를 추가하여:
- 남은 공간을 차지하되
- 너무 넓어지지 않도록 제한

## 📝 변경된 파일

### `app/(student)/contents/master-books/_components/HierarchicalFilter.tsx`
- 각 필터 항목에 `min-w-[XXXpx]` 추가
- 모든 select/input에 `w-full` 클래스 추가
- 제목 검색 필드에 `flex-1 max-w-[300px]` 추가

## 🎯 기대 효과

1. **안정적인 레이아웃**: 드롭다운 선택 시 레이아웃이 흔들리지 않음
2. **일관된 UI**: 모든 필터 항목이 일정한 최소 너비를 유지
3. **향상된 UX**: 사용자가 필터를 조작할 때 시각적 안정성 확보
4. **반응형 대응**: 작은 화면에서도 적절하게 배치

## 📌 적용된 최소 너비

- **개정교육과정**: `min-w-[160px]` (텍스트가 긴 편)
- **교과/과목**: `min-w-[140px]`
- **출판사/플랫폼**: `min-w-[140px]`
- **제목 검색**: `min-w-[200px] flex-1 max-w-[300px]` (유연한 너비)

## 🔄 반응형 고려사항

- `flex-wrap`을 유지하여 작은 화면에서는 자동으로 줄바꿈
- 각 필터 항목의 최소 너비가 모바일에서도 적절하게 표시되도록 설정
- 제목 검색 필드가 유연하게 남은 공간을 차지
