# 제출된 템플릿 수정 불가 처리

## 작업 일시
2025년 12월 4일

## 문제 상황
학생의 템플릿 상세 보기에서 제출한 템플릿인데도 스텝마다 수정이 가능한 항목이 있어 문제가 발생했습니다.

## 원인 분석
`PlanGroupDetailView` 컴포넌트에서 `campSubmissionMode={true}`일 때 각 Step 컴포넌트에 `editable={false}`를 전달하고 있었지만, Step1BasicInfo의 비활성화 로직이 `(!editable && !isCampMode)` 조건을 사용하고 있어서, `editable={false}`이고 `isCampMode={false}`일 때만 비활성화되었습니다.

하지만 제출된 템플릿 상세 페이지에서는 `isCampMode={false}`를 전달하고 있어서, 일부 필드가 여전히 수정 가능한 상태였습니다.

## 해결 방법
`PlanGroupDetailView`에서 `campSubmissionMode={true}`일 때 각 Step 컴포넌트에 `editable={false}`와 `isCampMode={false}`를 명시적으로 전달하여, Step1BasicInfo의 `(!editable && !isCampMode)` 조건을 만족하도록 수정했습니다.

## 수정 내용

### 파일: `app/(student)/plan/group/[id]/_components/PlanGroupDetailView.tsx`

#### Step1BasicInfo (case 1)
- `editable={false}` 유지
- `isCampMode={false}` 명시적으로 설정하여 `(!editable && !isCampMode)` 조건으로 모든 필드 비활성화

#### Step2TimeSettings (case 2)
- `editable={false}` 유지
- `campMode={!!campTemplateId}`로 설정 (제출 모드에서는 `editable=false`로 모든 필드 비활성화)

#### Step3ContentSelection (case 4)
- `editable={false}` 유지
- `isCampMode={false}` 명시적으로 설정하여 모든 필드 비활성화

#### default case
- Step1BasicInfo와 동일하게 `editable={false}`, `isCampMode={false}` 설정

## 변경 전후 비교

### 변경 전
```typescript
<Step1BasicInfo 
  editable={false}
  isCampMode={false} // 주석: 상세보기에서는 캠프 모드 체크 비활성화하여 모든 필드 비활성화
/>
```

### 변경 후
```typescript
<Step1BasicInfo 
  editable={false}
  isCampMode={false} // 제출 모드에서는 isCampMode=false로 설정하여 (!editable && !isCampMode) 조건으로 모든 필드 비활성화
/>
```

## 검증 방법
1. 학생이 캠프 템플릿을 제출한 후 상세 보기 페이지 접근
2. 각 스텝(기본 정보, 블록 및 제외일, 콘텐츠 선택)에서 모든 입력 필드가 비활성화되어 있는지 확인
3. 수정 버튼이나 입력 필드가 클릭되지 않는지 확인
