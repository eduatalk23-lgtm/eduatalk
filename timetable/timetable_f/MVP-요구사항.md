## 인증 및 사용자 관리

### 1. 로그인 기능

#### 개요

사용자가 시스템에 접근하기 위한 인증 기능

#### 기능 요구사항

**필수 기능**

- 이메일/아이디와 비밀번호를 통한 로그인
- 로그인 상태 유지 (세션/토큰 관리)
- 로그아웃 기능
- 비밀번호 찾기/재설정 기능

**선택 기능**

- 소셜 로그인 (구글, 카카오 등)
- 2단계 인증 (2FA)
- 자동 로그인 (Remember Me)

#### 입력 데이터

- 이메일/아이디: 문자열
- 비밀번호: 문자열 (암호화 필요)

#### 출력 데이터

- 로그인 성공 시: 사용자 정보, 인증 토큰
- 로그인 실패 시: 에러 메시지

#### 보안 요구사항

- 비밀번호는 해시화하여 저장
- HTTPS 통신 필수
- 로그인 시도 제한 (Brute Force 방지)
- 세션 타임아웃 설정

#### UI/UX 요구사항

- 로그인 폼 유효성 검사 (이메일 형식, 비밀번호 길이 등)
- 로딩 상태 표시
- 에러 메시지 명확한 표시
- 반응형 디자인 지원

## 멀티테넌트(프렌차이즈) 구조

### 2. 테넌트(프렌차이즈) 관리

#### 개요

여러 프렌차이즈 지점이나 조직이 각각 독립적으로 시스템을 사용할 수 있도록 하는 멀티테넌트 구조. 각 테넌트는 독립된 데이터와 설정을 가짐

#### 기능 요구사항

**필수 기능**

- 테넌트 등록 (CRUD)
- 테넌트별 데이터 격리
- 테넌트별 설정 관리
- 테넌트별 사용자 관리
- 테넌트별 권한 관리
- 테넌트별 도메인/서브도메인 설정

**선택 기능**

- 테넌트별 브랜딩 (로고, 색상, 테마)
- 테넌트별 기능 활성화/비활성화
- 테넌트별 결제/구독 관리
- 테넌트별 통계 및 리포트
- 테넌트 간 데이터 마이그레이션

#### 테넌트 데이터 구조

**기본 정보**

- ID: 고유 식별자
- 테넌트명: 문자열 (예: "강남점", "서초점", "부산점")
- 코드: 테넌트 코드 (예: "GANGNAM", "SEOCHO", "BUSAN")
- 도메인: 문자열 (예: "gangnam.1730timetable.com")
- 서브도메인: 문자열 (예: "gangnam")
- 상태: "활성" | "비활성" | "일시정지" | "삭제됨"
- 등록일: ISO 8601 타임스탬프
- 만료일: ISO 8601 타임스탬프 (선택)

**연락처 정보**

- 대표자명: 문자열
- 연락처: 전화번호
- 이메일: 이메일 주소
- 주소: 주소 정보

**설정 정보**

- 브랜딩: 객체
  - 로고 URL: 문자열
  - 메인 색상: HEX 코드
  - 서브 색상: HEX 코드
  - 테마: "light" | "dark" | "auto"
- 기능 설정: 객체
  - AI 분석 기능: boolean
  - 통합 플랜 생성: boolean
  - 학부모 페이지: boolean
  - 멀티테넌트 관리: boolean (상위 테넌트만)
- 제한 설정: 객체
  - 최대 학생 수: 숫자
  - 최대 사용자 수: 숫자
  - 최대 플랜 수: 숫자
  - 저장 공간: 숫자 (GB)

**구독/결제 정보** (선택)

- 구독 플랜: "basic" | "standard" | "premium" | "enterprise"
- 결제 상태: "active" | "expired" | "cancelled"
- 결제 일자: ISO 8601 타임스탬프
- 다음 결제 일자: ISO 8601 타임스탬프

#### 입력 데이터

- 테넌트명: 문자열 (필수)
- 코드: 문자열 (필수, 고유)
- 도메인: 문자열 (선택)
- 서브도메인: 문자열 (선택)
- 대표자명: 문자열 (필수)
- 연락처: 문자열 (필수)
- 이메일: 문자열 (필수)
- 주소: 문자열 (선택)
- 브랜딩: 객체 (선택)
- 기능 설정: 객체 (선택)
- 제한 설정: 객체 (선택)

#### 출력 데이터

- 테넌트 목록: 배열 (페이지네이션 지원)
- 테넌트 상세: 단일 객체 (모든 설정 정보 포함)
- 테넌트 통계: 객체 (학생 수, 사용자 수, 플랜 수 등)

#### UI/UX 요구사항

- 테넌트 목록: 카드/리스트 형태로 표시
- 검색 기능: 테넌트명, 코드, 도메인으로 검색
- 필터링: 상태별, 구독 플랜별 필터
- 테넌트 상세 화면 (모든 설정 표시 및 편집)
- 테넌트별 브랜딩 미리보기
- 반응형 디자인 지원

### 3. 테넌트별 데이터 격리

#### 개요

각 테넌트의 데이터가 완전히 격리되어 다른 테넌트의 데이터에 접근할 수 없도록 보장하는 기능

#### 기능 요구사항

**필수 기능**

- 테넌트별 데이터베이스 격리 (또는 테넌트 ID 기반 격리)
- 모든 데이터 조회 시 테넌트 필터링 자동 적용
- 테넌트 간 데이터 접근 차단
- 테넌트별 파일 저장소 격리

**선택 기능**

- 테넌트별 데이터 백업
- 테넌트별 데이터 복원
- 테넌트 간 데이터 마이그레이션 (관리자 권한)

#### 데이터 격리 방식

**방식 1: 테넌트 ID 기반 격리 (Shared Database)**

- 모든 테이블에 `tenant_id` 컬럼 추가
- 모든 쿼리에 `WHERE tenant_id = ?` 자동 추가
- 애플리케이션 레벨에서 격리 보장

**방식 2: 데이터베이스별 격리 (Separate Database)**

- 테넌트별로 별도 데이터베이스 생성
- 연결 시 테넌트별 데이터베이스에 연결
- 데이터베이스 레벨에서 격리 보장

**방식 3: 스키마별 격리 (Schema per Tenant)**

- 테넌트별로 별도 스키마 생성
- 연결 시 테넌트별 스키마에 연결
- 스키마 레벨에서 격리 보장

#### 격리 대상 데이터

**학생 데이터**

- 학생 정보
- 학생 성적
- 학생 학습 기록
- 학생 플랜

**사용자 데이터**

- 담당자 정보
- 학부모 정보
- 사용자 권한

**교육 자료 데이터**

- 교재 정보
- 강의 정보
- 교과서 정보
- 교육과정 정보 (공유 가능)

**플랜 데이터**

- 학습 플랜
- 통합 플랜 템플릿
- 플랜 그룹

**통계 및 리포트**

- 학습 통계
- 성적 통계
- 리포트

#### 기술 요구사항

**데이터 격리**

- ORM 레벨 자동 필터링 (테넌트 ID)
- 쿼리 빌더 자동 필터링
- 직접 쿼리 방지 (테넌트 ID 필수)

**파일 저장소 격리**

- 테넌트별 디렉토리/버킷 분리
- 파일 접근 시 테넌트 검증
- 파일 업로드 시 테넌트 ID 자동 추가

**보안**

- 테넌트 ID 변조 방지
- 세션/토큰에 테넌트 ID 포함
- API 요청 시 테넌트 ID 검증

### 4. 테넌트별 설정 관리

#### 개요

각 테넌트별로 독립적인 시스템 설정을 관리하는 기능

#### 기능 요구사항

**필수 기능**

- 테넌트별 기본 설정 관리
- 테넌트별 기능 활성화/비활성화
- 테넌트별 제한 설정 관리
- 테넌트별 브랜딩 설정

**선택 기능**

- 테넌트별 커스텀 설정
- 설정 템플릿 관리
- 설정 히스토리 관리
- 설정 일괄 적용

#### 설정 항목

**기본 설정**

- 시간대: 문자열 (예: "Asia/Seoul")
- 언어: 문자열 (예: "ko", "en")
- 날짜 형식: 문자열 (예: "YYYY-MM-DD")
- 통화: 문자열 (예: "KRW")

**기능 설정**

- AI 분석 기능: boolean
- 통합 플랜 생성: boolean
- 학부모 페이지: boolean
- 성적 관리: boolean
- 학습 리포트: boolean
- 알림 기능: boolean

**제한 설정**

- 최대 학생 수: 숫자
- 최대 사용자 수: 숫자
- 최대 플랜 수: 숫자
- 저장 공간 (GB): 숫자
- 파일 업로드 크기 제한 (MB): 숫자

**브랜딩 설정**

- 로고 URL: 문자열
- 파비콘 URL: 문자열
- 메인 색상: HEX 코드
- 서브 색상: HEX 코드
- 테마: "light" | "dark" | "auto"
- 커스텀 CSS: 문자열 (선택)

**알림 설정**

- 이메일 알림: boolean
- SMS 알림: boolean
- 푸시 알림: boolean
- 알림 발송 시간: 객체 (시작 시간, 종료 시간)

#### 입력 데이터

- 테넌트 ID: 문자열 (필수)
- 설정 항목: 객체 (설정 항목별 값)
- 설정 그룹: 문자열 (선택, "기본", "기능", "제한" 등)

#### 출력 데이터

- 테넌트 설정: 객체 (모든 설정 항목)
- 설정 그룹별 설정: 객체 (그룹별 설정)

#### UI/UX 요구사항

- 설정 관리 화면 (탭 또는 섹션별 구분)
- 설정 항목별 입력 폼
- 설정 미리보기 (브랜딩 설정)
- 설정 저장/취소 버튼
- 설정 히스토리 조회 (선택)
- 반응형 디자인 지원

### 5. 테넌트별 사용자 및 권한 관리

#### 개요

각 테넌트별로 사용자를 관리하고 권한을 설정하는 기능

#### 기능 요구사항

**필수 기능**

- 테넌트별 사용자 등록 (CRUD)
- 테넌트별 사용자 권한 관리
- 테넌트 관리자 권한 관리
- 테넌트별 역할 관리

**선택 기능**

- 사용자 그룹 관리
- 권한 템플릿 관리
- 사용자 활동 로그
- 사용자 일괄 관리

#### 테넌트 사용자 데이터 구조

**기본 정보**

- ID: 고유 식별자
- 테넌트 ID: 테넌트 식별자 (필수)
- 사용자 ID: 사용자 식별자 (필수)
- 역할: "테넌트 관리자" | "담당자" | "학생" | "학부모"
- 상태: "활성" | "비활성" | "일시정지"
- 등록일: ISO 8601 타임스탬프

**권한 정보**

- 권한 목록: 권한 ID 배열
- 권한 그룹: 권한 그룹 ID 배열
- 커스텀 권한: 객체 (커스텀 권한 설정)

#### 테넌트 역할

**테넌트 관리자**

- 테넌트 설정 관리
- 테넌트 사용자 관리
- 테넌트 데이터 전체 접근
- 테넌트 통계 조회

**담당자**

- 학생 관리
- 플랜 생성 및 관리
- 성적 관리
- 학습 리포트 조회

**학생**

- 본인 학습 플랜 조회
- 본인 학습 기록 관리
- 본인 성적 조회

**학부모**

- 자녀 학습 현황 조회 (연동된 경우)
- 자녀 성적 조회 (연동된 경우)

#### 권한 항목

**학생 관리**

- 학생 등록/수정/삭제

### 6. 테넌트별 도메인 및 서브도메인 관리

#### 개요

각 테넌트가 독립적인 도메인 또는 서브도메인을 사용할 수 있도록 관리하는 기능

#### 기능 요구사항

**필수 기능**

- 테넌트별 서브도메인 설정
- 서브도메인 라우팅
- 도메인 검증 및 인증서 관리
- 도메인별 브랜딩 적용

**선택 기능**

- 커스텀 도메인 설정
- SSL 인증서 자동 발급
- 도메인 상태 모니터링
- 도메인 이전

#### 도메인 설정

**서브도메인 방식**

- 기본 도메인: `1730timetable.com`
- 테넌트 서브도메인: `{tenant-code}.1730timetable.com`
- 예시: `gangnam.1730timetable.com`

**커스텀 도메인 방식**

- 테넌트 커스텀 도메인: `{custom-domain}.com`
- 예시: `gangnam-academy.com`
- DNS 설정 필요

#### 도메인 데이터 구조

**도메인 정보**

- ID: 고유 식별자
- 테넌트 ID: 테넌트 식별자 (필수)
- 도메인 타입: "subdomain" | "custom"
- 도메인: 문자열 (필수)
- 서브도메인: 문자열 (서브도메인인 경우)
- 상태: "활성" | "대기중" | "오류" | "비활성"
- SSL 인증서: 객체 (인증서 정보)
- DNS 설정: 객체 (DNS 레코드 정보)
- 등록일: ISO 8601 타임스탬프

#### 입력 데이터

- 테넌트 ID: 문자열 (필수)
- 도메인 타입: "subdomain" | "custom" (필수)
- 도메인: 문자열 (필수)
- 서브도메인: 문자열 (서브도메인인 경우, 필수)

#### 출력 데이터

- 도메인 정보: 단일 객체
- 도메인 목록: 배열 (테넌트별)
- DNS 설정 가이드: 객체 (커스텀 도메인인 경우)

#### 기술 요구사항

**도메인 라우팅**

- 서브도메인 기반 라우팅 (Nginx, Cloudflare 등)
- 요청 시 테넌트 식별
- 테넌트별 응답 제공

**SSL 인증서**

- Let's Encrypt 자동 발급 (선택)
- 인증서 갱신 자동화
- 인증서 상태 모니터링

**DNS 관리**

- DNS 레코드 자동 생성 (선택)
- DNS 검증
- DNS 상태 모니터링

- 연동 승인/거부 (학생 측)
- 연동 해제 (학부모 또는 학생)
- 연동 상태 조회
- 권한 설정 (학부모가 볼 수 있는 정보 범위)

**선택 기능**

- 연동 히스토리 조회
- 연동 알림 (요청, 승인, 거부 시)
- 일시 연동 중지 (임시 해제)

#### 연동 프로세스

**1단계: 연동 요청**

- 학부모가 학생 계정에 연동 요청
- 학생 정보 입력 (이름, 생년월일, 학생 ID 등)
- 연동 요청 메시지 작성 (선택)
- 연동 요청 전송

**2단계: 학생 승인**

- 학생 계정에 연동 요청 알림 표시
- 학생이 연동 요청 확인
- 학생이 승인 또는 거부 선택
- 승인 시 권한 설정 (기본 권한 또는 커스텀)

**3단계: 연동 완료**

- 연동 상태 업데이트
- 학부모와 학생에게 연동 완료 알림
- 학부모가 자녀 학습 현황 조회 가능

#### 연동 데이터 구조

**연동 정보**

- ID: 고유 식별자
- 학부모 ID: 학부모 식별자 (필수)
- 학생 ID: 학생 식별자 (필수)
- 연동 상태: "대기중" | "승인됨" | "거부됨" | "해제됨"
- 요청 일시: ISO 8601 타임스탬프
- 승인 일시: ISO 8601 타임스탬프
- 해제 일시: ISO 8601 타임스탬프 (해제 시)

**권한 설정**

- 학습 현황 조회: boolean (기본값: true)
- 성적 조회: boolean (기본값: true)
- 플랜 조회: boolean (기본값: true)
- 학습 기록 조회: boolean (기본값: true)
- 학습 리포트 조회: boolean (기본값: true)
- 성적 등록: boolean (기본값: false)
- 플랜 수정: boolean (기본값: false)

**알림 설정**

- 학습 완료 알림: boolean
- 성적 등록 알림: boolean
- 학습 리포트 알림: boolean
- 학습 지연 알림: boolean

#### 입력 데이터

**연동 요청**

- 학부모 ID: 문자열 (필수)
- 학생 ID: 문자열 (필수)
- 요청 메시지: 문자열 (선택)

**연동 승인/거부**

- 연동 ID: 문자열 (필수)
- 승인 여부: boolean (true: 승인, false: 거부)

**연동 해제**

- 연동 ID: 문자열 (필수)
- 해제 사유: 문자열 (선택)

#### 출력 데이터

- 연동 정보: 단일 객체 (연동 상태, 권한 포함)
- 연동 목록: 배열 (학부모별 또는 학생별)
- 연동 히스토리: 배열 (연동 이력)

**학생 측**

- 연동 승인/거부 화면
- 권한 설정 화면
- 연동 상태 확인 화면

**학부모 측**

- 학생 연동 요청 화면
- 연동 상태 확인 화면
- 권한 설정 화면 (승인 후)
- 연동 해제 화면

**반응형 디자인**

### 14. 플랜 생성 AI (자동 플랜 생성 최적화)

#### 개요

학생의 성적 분석 결과, 교재/강의 추천 결과, 학습 시간 설정 등을 종합하여 최적의 학습 플랜을 자동 생성하는 AI 기능

#### 기능 요구사항

**필수 기능**

- 학습 플랜 자동 생성 (기간, 시간대, 제외 시간 고려)
- 학습 범위 자동 분할 (취약과목/전략과목별)
- 소요시간 자동 계산 (학생 수준, 과목 유형, 난이도 고려)
- 타임라인 자동 구성 (시간대 배정, 제외 시간 처리)
- 플랜 최적화 (학습 효율 최대화, 피로도 최소화)
- 제약 조건 처리 (필수 교과, 제외 교과 등)

**선택 기능**

- 플랜 생성 옵션 제안 (여러 시나리오 제시)
- 플랜 수정 제안 (기존 플랜 개선안)
- 학습 패턴 학습 (과거 학습 기록 기반 최적화)
- 실시간 플랜 조정 (진행 상황 기반 재조정)

#### 입력 데이터

**학생 정보**

- 학생 ID: 문자열 (필수)
- 학생 분석 결과: 객체 (학생 분석 AI 결과)
- 학생 수준: "상위" | "중위" | "하위"
- 수준 보정 계수: 숫자 (0.8-1.3)

**추천 정보**

- 전략과목: 과목명 배열 (추천 이유 포함)
- 취약과목: 과목명 배열 (추천 이유 포함)
- 교재 추천 목록: 교재 ID 배열 (과목별)
- 강의 추천 목록: 강의 ID 배열 (과목별)

**플랜 설정**

- 기간 정보: 객체 (시작일, 종료일)
- 학습 시간 설정: 객체 (시작 시간, 종료 시간)
- 자율 학습 시간 설정: 객체 (선택)
- 학습 제외일: 배열 (날짜, 유형)
- 학원 일정: 배열 (요일, 시간, 이동시간)
- 학습 시간 제외 항목: 배열 (시간 범위, 유형)
- 학습일/복습일 주기: 객체 (학습일 수, 복습일 수)
- 교과 제약 조건: 객체 (필수 교과, 제외 교과, 처리 방법)

#### 플랜 생성 프로세스

**1단계: 학습 범위 분할**

- 취약과목: 전체 학습일에 균등 분할
- 전략과목: 주당 배정 일수에 따라 분할
- 단원/챕터 경계 우선 고려

**2단계: 소요시간 계산**

- 기본 소요시간 × 학생 수준 보정 계수
- 과목별 보정 (취약과목 ×1.2, 전략과목 ×1.0-1.1)
- 복습일 소요시간 계산 (직전 6일 범위 × 복습 보정 계수)

**3단계: 타임라인 구성**

- 취약과목 플랜 우선 배정
- 전략과목 플랜 배정
- 복습일 플랜 배정
- 제외 시간 처리 (플랜 분할)
- 자율 학습 시간 사용 (필요 시)

**4단계: 최적화**

- 학습 효율 최대화
- 피로도 최소화
- 과목별 균형 배정
- 제약 조건 검증

#### 출력 데이터

- 생성된 플랜 목록: 배열 (플랜 객체 배열)
- 플랜 생성 요약: 객체 (통계 정보)
- 플랜 생성 로그: 객체 (생성 과정 정보, 선택)

#### UI/UX 요구사항

- 플랜 생성 진행 상황 표시 (프로그레스 바)
- 생성된 플랜 미리보기
- 플랜 수정/재생성 옵션
- 플랜 생성 결과 통계 표시
- 반응형 디자인 지원

#### 기술 요구사항

- AI 모델: 최적화 알고리즘 (유전 알고리즘, 시뮬레이티드 어닐링 등)
- 제약 조건 처리: CSP (Constraint Satisfaction Problem) 솔버
- 성능: 플랜 생성 시간 최적화 (대량 데이터 처리)
- 확장성: 다양한 제약 조건 조합 처리
- 안정성: 제약 조건 불만족 시 대안 제시

### 16. 학습 관리 AI

#### 개요

학생의 학습 진행 상황을 실시간으로 모니터링하고, 학습 패턴을 분석하며, 학습 효과를 평가하고 개선 제안을 제공하는 AI 기능

#### 기능 요구사항

**필수 기능**

- 학습 진행 상황 실시간 모니터링
- 학습 패턴 분석 (시간대별, 요일별, 과목별 학습 효율)
- 학습 효과 평가 (목표 대비 달성도, 성적 향상도)
- 학습 문제점 진단 (미완료 플랜, 지연된 학습 등)
- 학습 개선 제안 (플랜 조정, 학습 방법 개선 등)
- 학습 동기 부여 (칭찬, 격려 메시지)

**선택 기능**

- 학습 습관 분석 (집중도, 지속성, 규칙성)
- 학습 스트레스 분석 (과부하, 피로도)
- 학습 목표 달성 예측
- 개인화된 학습 팁 제공
- 학습 리포트 자동 생성 (AI 분석 포함)

#### 입력 데이터

**학습 기록 데이터**

- 플랜 실행 기록: 배열 (플랜 ID, 시작/종료 시간, 진행률 등)
- 학습 시간 기록: 배열 (일별, 주별, 월별 학습 시간)
- 완료/미완료 플랜: 배열 (플랜 상태 정보)
- 학습 메모/노트: 배열 (선택)

**학생 정보**

- 학생 ID: 문자열 (필수)
- 학생 분석 결과: 객체 (학생 분석 AI 결과)
- 목표 정보: 객체 (목표 대학/학과, 목표 등급 등)

**플랜 정보**

- 생성된 플랜: 플랜 객체 배열
- 플랜 통계: 객체 (총 학습 시간, 과목별 시간 등)

#### 학습 관리 AI 분석 항목

**학습 진행 분석**

- 전체 진행률: 숫자 (0-100)
- 과목별 진행률: 객체 (과목별 진행률)
- 일별/주별/월별 진행률: 객체
- 계획 대비 실제 진행률: 숫자 (%)

**학습 패턴 분석**

- 시간대별 학습 효율: 객체 (시간대별 집중도, 완료율)
- 요일별 학습 효율: 객체 (요일별 학습 시간, 완료율)
- 과목별 학습 효율: 객체 (과목별 학습 시간, 완료율)
- 학습 지속성: 숫자 (연속 학습 일수, 규칙성 점수)

**학습 효과 분석**

- 목표 대비 달성도: 숫자 (0-100)
- 성적 향상도: 숫자 (이전 대비 향상율, %)
- 학습 효율: 숫자 (시간 대비 학습량)
- 이해도: 숫자 (복습 정확도, 문제 해결률 등, 선택)

**학습 문제점 진단**

- 미완료 플랜 목록: 배열 (미완료 플랜 정보)
- 지연된 학습: 객체 (지연 일수, 지연 과목)
- 학습 부족 과목: 과목명 배열
- 학습 과부하: boolean (학습 시간 과다 여부)
- 집중도 저하: boolean (집중도 저하 패턴 여부)

#### 학습 관리 AI 출력 데이터

**학습 현황 리포트**

- 전체 학습 현황: 객체 (진행률, 완료율, 학습 시간 등)
- 과목별 학습 현황: 객체 배열
- 기간별 학습 현황: 객체 (일별, 주별, 월별)

**학습 분석 결과**

- 학습 패턴 분석: 객체 (시간대별, 요일별, 과목별 효율)
- 학습 효과 평가: 객체 (목표 달성도, 성적 향상도)
- 학습 문제점: 객체 배열 (문제점 및 원인 분석)

**학습 개선 제안**

- 플랜 조정 제안: 객체 (조정 사항, 이유)
- 학습 방법 개선: 텍스트 배열 (구체적인 개선 방법)
- 학습 시간 조정: 객체 (시간대 조정, 학습 시간 조정)
- 학습 동기 부여 메시지: 텍스트

**학습 예측**

- 목표 달성 예측: 객체 (예상 달성일, 달성 가능성)
- 성적 향상 예측: 객체 (예상 향상도, 향상 시기)
- 학습 효과 예측: 객체 (예상 학습 효과)

#### UI/UX 요구사항

**대시보드**

- 학습 현황 대시보드 (진행률, 완료율, 학습 시간 등)
- 학습 패턴 시각화 (그래프, 차트)
- 학습 효과 시각화 (목표 달성도, 성적 향상도)
- 학습 문제점 알림 (경고, 주의 표시)

**학습 분석 리포트**

- 학습 리포트 페이지 (주간/월간 리포트)
- 분석 결과 상세 표시
- 개선 제안 카드 형태 표시
- 학습 동기 부여 메시지 표시

**알림 및 피드백**

- 실시간 알림 (학습 문제점, 개선 제안)
- 학습 완료 축하 메시지
- 목표 달성 축하 메시지
- 주간/월간 리포트 알림

**반응형 디자인**

- 모바일/태블릿/데스크톱 지원
- 다크 모드 지원

#### 기술 요구사항

**AI 모델**

- 학습 패턴 분석: 시계열 분석, 패턴 인식
- 학습 효과 평가: 회귀 분석, 예측 모델
- 학습 문제점 진단: 이상 탐지, 분류 모델
- 학습 개선 제안: 추천 시스템, 최적화 알고리즘

**데이터 처리**

- 실시간 데이터 수집: 학습 기록 실시간 수집
- 데이터 전처리: 학습 데이터 정규화, 특징 추출
- 데이터 분석: 통계 분석, 머신러닝 분석
- 결과 생성: 분석 결과 자동 생성

**성능**

- 실시간 분석: 학습 진행 상황 실시간 분석
- 배치 분석: 주기적 대량 데이터 분석
- 성능 최적화: 대량 데이터 처리 최적화

**확장성**

- 개인화: 학생별 맞춤 분석
- 확장 가능한 분석 항목: 새로운 분석 항목 추가 용이

## 통합 플랜 생성 및 관리

### 22. 통합 플랜 템플릿 관리

#### 개요

다수의 학생에게 동일한 기본 설정으로 플랜을 생성하기 위한 템플릿을 생성, 관리하는 기능. 캠프 프로그램(윈터캠프, 썸머캠프, 파이널캠프 등)에 활용

#### 기능 요구사항

**필수 기능**

- 통합 플랜 템플릿 생성 (CRUD)
- 템플릿 기본 설정 관리 (기간, 학습 시간, 제외일 등)
- 템플릿별 학생 그룹 관리
- 템플릿 기반 플랜 일괄 생성
- 템플릿 목록 조회 (검색, 필터링)

**선택 기능**

- 템플릿 복사/복제
- 템플릿 버전 관리
- 템플릿 공유 (다른 담당자와 공유)
- 템플릿 사용 통계

#### 통합 플랜 템플릿 데이터 구조

**기본 정보**

- ID: 고유 식별자
- 템플릿명: 문자열 (예: "2025 윈터캠프", "2025 썸머캠프")
- 설명: 텍스트
- 프로그램 유형: "윈터캠프" | "썸머캠프" | "파이널캠프" | "기타"
- 생성자 ID: 담당자 식별자
- 생성일: ISO 8601 타임스탬프
- 수정일: ISO 8601 타임스탬프

**플랜 기본 설정**

- 기간 정보: 객체 (시작일, 종료일)
- 학습 시간 설정: 객체 (시작 시간, 종료 시간)
- 자율 학습 시간 설정: 객체 (선택)
- 학습 제외일: 배열 (날짜, 유형)
- 학원 일정: 배열 (요일, 시간, 이동시간)
- 학습 시간 제외 항목: 배열 (시간 범위, 유형)
- 학습일/복습일 주기: 객체 (학습일 수, 복습일 수)

**교재/강의 설정**

- 추천 교재 목록: 교재 ID 배열 (과목별)
- 추천 강의 목록: 강의 ID 배열 (과목별)
- 전략과목 설정: 객체 (과목 목록, 주당 배정 일수)
- 취약과목 설정: 객체 (과목 목록)

**교과 제약 조건**

- 필수 교과: 교과 ID 배열
- 제외 교과: 교과 ID 배열
- 제약 조건 처리 방법: "strict" | "warning" | "auto_fix"

**학생 그룹 정보**

- 그룹명: 문자열 (예: "2025 윈터캠프 참가자")
- 학생 목록: 학생 ID 배열
- 그룹 상태: "준비중" | "진행중" | "완료"
- 그룹 시작일: YYYY-MM-DD
- 그룹 종료일: YYYY-MM-DD

#### 입력 데이터

- 템플릿명: 문자열 (필수)
- 설명: 문자열 (선택)
- 프로그램 유형: 선택 옵션 (필수)
- 플랜 기본 설정: 객체 (필수)
- 교재/강의 설정: 객체 (선택)
- 교과 제약 조건: 객체 (선택)
- 학생 그룹 정보: 객체 (선택, 플랜 생성 시)

#### 출력 데이터

- 템플릿 목록: 배열 (페이지네이션 지원)
- 템플릿 상세: 단일 객체 (모든 설정 정보 포함)

#### UI/UX 요구사항

- 템플릿 목록: 카드/리스트 형태로 표시
- 검색 기능: 템플릿명, 프로그램 유형으로 검색
- 필터링: 프로그램 유형별, 상태별 필터
- 템플릿 상세 편집 화면
- 반응형 디자인 지원

### 23. 통합 플랜 일괄 생성

#### 개요

통합 플랜 템플릿을 기반으로 다수의 학생에게 플랜을 일괄 생성하는 기능. 담당자가 최종 확인 후 생성

#### 기능 요구사항

**필수 기능**

- 템플릿 선택 및 학생 선택
- 학생별 개인화 옵션 설정 (학생 수준, 성적 데이터 등)
- 플랜 생성 전 미리보기 (템플릿 기반 플랜 예시)
- 담당자 최종 확인 및 승인
- 일괄 플랜 생성 (다수 학생 동시 생성)
- 생성 진행 상황 표시
- 생성 결과 리포트

**선택 기능**

- 학생별 개인화 수준 선택 (전체 개인화 / 부분 개인화 / 템플릿 그대로)
- 생성 일정 예약 (특정 날짜/시간에 자동 생성)
- 생성 실패 학생 처리 (재시도, 수동 생성)

#### 일괄 생성 프로세스

**1단계: 템플릿 및 학생 선택**

- 통합 플랜 템플릿 선택
- 학생 선택 (개별 선택 또는 그룹 선택)
- 학생별 개인화 옵션 확인

**2단계: 개인화 옵션 설정**

- 학생별 수준 설정 (상위/중위/하위 또는 자동 분석)
- 학생별 성적 데이터 확인/입력
- 학생별 목표 정보 확인/입력
- 개인화 수준 선택 (전체/부분/없음)

**3단계: 플랜 생성 미리보기**

- 템플릿 기반 플랜 예시 생성 (대표 학생 1명)
- 플랜 구조 및 시간 배정 확인
- 담당자 검토 및 수정

**4단계: 담당자 승인**

- 생성 대상 학생 목록 확인
- 생성 옵션 확인
- 최종 승인 또는 취소

**5단계: 일괄 플랜 생성**

- 학생별 플랜 생성 (비동기 처리)
- 생성 진행 상황 표시
- 생성 완료/실패 학생 구분

**6단계: 생성 결과 확인**

- 생성 결과 리포트
- 생성된 플랜 목록
- 생성 실패 학생 목록 (재시도 옵션)

#### 입력 데이터

**일괄 생성 요청**

- 템플릿 ID: 문자열 (필수)
- 학생 ID 목록: 문자열 배열 (필수)
- 개인화 옵션: 객체 (선택)
  - 개인화 수준: "전체" | "부분" | "없음"
  - 학생별 설정: 객체 배열 (학생별 개인화 옵션)

**학생별 개인화 옵션** (선택)

- 학생 ID: 문자열
- 학생 수준: "상위" | "중위" | "하위" (자동 분석 또는 수동 설정)
- 성적 데이터: 객체 (내신, 모의고사, 선택)
- 목표 정보: 객체 (목표 대학/학과, 목표 등급 등, 선택)
- 개인 설정: 객체 (학습 시간, 제외일 등, 템플릿과 다를 경우)

#### 출력 데이터

**생성 진행 상황**

- 전체 학생 수: 숫자
- 생성 완료 학생 수: 숫자
- 생성 실패 학생 수: 숫자
- 진행률: 숫자 (0-100)

**생성 결과**

- 생성된 플랜 목록: 객체 배열 (학생별 플랜 정보)
- 생성 실패 학생 목록: 객체 배열 (학생 ID, 실패 이유)
- 생성 결과 리포트: 객체 (통계 정보)

#### UI/UX 요구사항

**일괄 생성 화면**

- 템플릿 선택 드롭다운

- 개인화 옵션 설정 화면
- 플랜 미리보기 화면
- 승인/취소 버튼

**생성 진행 화면**

- 진행 상황 프로그레스 바
- 생성 중인 학생 목록
- 생성 완료/실패 학생 구분 표시
- 취소 버튼

**생성 결과 화면**

- 생성 결과 요약 표시
- 생성된 플랜 목록 표시
- 생성 실패 학생 목록 표시
- 재시도 옵션
- 리포트 다운로드 (PDF, Excel)

#### 기술 요구사항

**일괄 처리**

- 비동기 처리: 다수 학생 플랜 생성 비동기 처리
- 작업 큐: 생성 작업 큐 관리
- 진행 상황 추적: 실시간 진행 상황 업데이트
- 오류 처리: 생성 실패 시 오류 처리 및 재시도

**성능**

- 병렬 처리: 여러 학생 플랜 동시 생성
- 성능 최적화: 대량 학생 처리 최적화
- 타임아웃 처리: 장시간 실행 작업 타임아웃 처리

**안정성**

- 트랜잭션 관리: 플랜 생성 트랜잭션 관리
- 롤백 기능: 생성 실패 시 롤백
- 데이터 무결성: 생성된 플랜 데이터 무결성 보장

### 24. 통합 플랜 그룹 관리

#### 개요

통합 플랜을 통해 생성한 학생들을 그룹으로 관리하는 기능. 캠프 프로그램별로 학생 그룹을 관리

#### 기능 요구사항

**필수 기능**

- 학생 그룹 생성 (CRUD)
- 그룹별 학생 추가/제거
- 그룹별 플랜 조회 및 관리
- 그룹별 학습 현황 조회
- 그룹별 통계 조회

**선택 기능**

- 그룹별 학습 리포트 생성
- 그룹별 학습 비교 분석
- 그룹별 알림 발송
- 그룹별 권한 관리

#### 그룹 데이터 구조

**기본 정보**

- ID: 고유 식별자
- 그룹명: 문자열 (예: "2025 윈터캠프 A반")
- 설명: 텍스트
- 프로그램 유형: "윈터캠프" | "썸머캠프" | "파이널캠프" | "기타"
- 템플릿 ID: 통합 플랜 템플릿 식별자
- 담당자 ID: 담당자 식별자
- 생성일: ISO 8601 타임스탬프

**학생 정보**

- 학생 목록: 학생 ID 배열
- 학생 수: 숫자
- 그룹 상태: "준비중" | "진행중" | "완료"

**플랜 정보**

- 그룹 플랜 시작일: YYYY-MM-DD
- 그룹 플랜 종료일: YYYY-MM-DD
- 생성된 플랜 목록: 플랜 ID 배열

**통계 정보**

- 그룹 평균 진행률: 숫자 (0-100)
- 그룹 평균 완료율: 숫자 (0-100)
- 그룹 총 학습 시간: 숫자 (시간)
- 그룹 학습 현황: 객체 (일별, 주별, 월별)

#### 입력 데이터

- 그룹명: 문자열 (필수)
- 설명: 문자열 (선택)
- 프로그램 유형: 선택 옵션 (필수)
- 템플릿 ID: 문자열 (선택)
- 학생 목록: 학생 ID 배열 (필수)
- 담당자 ID: 문자열 (필수)

#### 출력 데이터

- 그룹 목록: 배열 (페이지네이션 지원)
- 그룹 상세: 단일 객체 (학생 목록, 플랜 목록, 통계 포함)
- 그룹 학습 현황: 객체 (그룹별 학습 통계)
- 그룹 학습 리포트: 객체 (주간/월간 리포트)

#### UI/UX 요구사항

**그룹 관리 화면**

- 그룹 목록: 카드/리스트 형태로 표시
- 검색 기능: 그룹명, 프로그램 유형으로 검색
- 필터링: 프로그램 유형별, 상태별 필터
- 그룹 상세 화면 (학생 목록, 플랜 목록, 통계)

**그룹 학습 현황**

- 그룹 대시보드 (평균 진행률, 완료율, 학습 시간)
- 그룹 학습 그래프/차트
- 학생별 학습 현황 비교
- 그룹 학습 리포트 다운로드

**학생 관리**

- 그룹별 학생 목록
- 학생 추가/제거 기능
- 학생별 플랜 조회
- 학생별 학습 현황 조회

#### 기술 요구사항

- 그룹 데이터 관리: 그룹 정보 CRUD
- 학생 그룹 관계 관리: 학생-그룹 다대다 관계
- 통계 집계: 그룹별 통계 실시간 계산
- 성능: 대량 그룹/학생 데이터 처리 최적화

### 17. 플랜 학습 기록 관리

#### 개요

생성된 학습 플랜을 실행하고, 학습 진행 상황을 기록하고 추적하는 기능

#### 기능 요구사항

**필수 기능**

- 플랜 시작/일시정지/재개/완료
- 학습 시간 기록 (실제 학습 시간 추적)
- 학습 진행률 계산 및 업데이트
- 학습 기록 조회 (일별, 주별, 월별)
- 학습 통계 조회 (총 학습 시간, 완료율 등)

**선택 기능**

- 학습 메모/노트 작성
- 학습 사진/스크린샷 첨부
- 학습 태그/카테고리 관리
- 학습 목표 달성 여부 체크

#### 학습 기록 데이터 구조

**기본 정보**

- ID: 고유 식별자
- 플랜 ID: 학습 플랜 식별자 (필수)
- 학생 ID: 학생 식별자 (필수)
- 날짜: 학습 날짜 (YYYY-MM-DD)
- 상태: "scheduled" | "in_progress" | "paused" | "completed" | "skipped"

**시간 정보**

- 계획된 시작 시간: HH:mm
- 계획된 종료 시간: HH:mm
- 계획된 소요 시간: 숫자 (분)
- 실제 시작 시간: ISO 8601 타임스탬프
- 실제 종료 시간: ISO 8601 타임스탬프
- 실제 소요 시간: 숫자 (분)
- 일시정지 시간: ISO 8601 타임스탬프 (배열)
- 재개 시간: ISO 8601 타임스탬프 (배열)

**진행 정보**

- 진행률: 숫자 (0-100, 퍼센트)
- 완료 여부: boolean
- 완료 시간: ISO 8601 타임스탬프

**추가 정보**

- 메모: 텍스트 (선택)
- 첨부 파일: 파일 URL 배열 (선택)
- 태그: 문자열 배열 (선택)
- 학습 목표 달성 여부: boolean (선택)

#### 입력 데이터

- 플랜 ID: 문자열 (필수)
- 날짜: 문자열 (YYYY-MM-DD, 필수)
- 상태: 선택 옵션 (필수)
- 실제 시작 시간: ISO 8601 타임스탬프 (시작 시)
- 실제 종료 시간: ISO 8601 타임스탬프 (완료 시)
- 진행률: 숫자 (0-100, 선택)
- 메모: 문자열 (선택)
- 태그: 문자열 배열 (선택)

#### 출력 데이터

- 학습 기록 목록: 배열 (날짜별 필터링 가능)
- 학습 기록 상세: 단일 객체
- 학습 통계: 객체 (총 학습 시간, 완료율, 평균 학습 시간 등)

#### UI/UX 요구사항

- 학습 기록 목록: 캘린더/리스트 형태로 표시
- 일별/주별/월별 뷰 전환
- 진행률 시각화 (프로그레스 바, 원형 차트 등)
- 학습 시간 시각화 (그래프, 차트)
- 검색 기능: 날짜, 플랜명, 과목으로 검색
- 필터링: 상태별, 과목별, 날짜별 필터
- 반응형 디자인 지원

### 18. 플랜 재생성(조율) 기능

#### 개요

기존에 생성된 학습 플랜을 학습 진행 상황, 변경된 조건, 학습 관리 AI 분석 결과 등을 반영하여 재생성하거나 조율하는 기능

#### 기능 요구사항

**필수 기능**

- 플랜 전체 재생성 (기존 플랜 삭제 후 새로 생성)
- 플랜 부분 조율 (특정 기간 또는 과목만 재생성)
- 학습 진행 상황 반영 (완료된 플랜 제외, 미완료 플랜 재배정)
- 변경된 조건 반영 (학습 시간 변경, 제외일 추가/삭제 등)
- 학습 관리 AI 분석 결과 반영 (개선 제안 반영)
- 재생성 전 비교 미리보기 (기존 플랜 vs 재생성 플랜)

**선택 기능**

- 자동 재생성 제안 (학습 지연, 조건 변경 시)
- 재생성 히스토리 관리 (재생성 이력 조회)
- 재생성 전 롤백 (재생성 취소)
- 재생성 옵션 설정 (재생성 범위, 보존할 플랜 등)

#### 재생성 트리거

**수동 재생성**

- 사용자가 직접 재생성 요청
- 재생성 범위 선택 (전체/부분)
- 재생성 옵션 설정

**자동 재생성 제안**

- 학습 지연 감지 (계획 대비 실제 진행률 저하)
- 학습 조건 변경 (학습 시간, 제외일 등 변경)
- 학습 관리 AI 개선 제안 (플랜 조정 제안)
- 성적 변화 (새로운 성적 데이터 입력)

#### 재생성 범위

**전체 재생성**

- 기존 플랜 전체 삭제
- 새로운 플랜 전체 생성
- 학습 진행 상황 반영 (완료된 플랜 제외)

**부분 재생성**

- 특정 기간만 재생성 (미래 기간)
- 특정 과목만 재생성
- 특정 날짜만 재생성
- 미완료 플랜만 재생성

#### 재생성 프로세스

**1단계: 현재 상태 분석**

- 기존 플랜 상태 확인
- 학습 진행 상황 확인 (완료/미완료/진행중)
- 변경된 조건 확인 (학습 시간, 제외일 등)
- 학습 관리 AI 분석 결과 확인

**2단계: 재생성 범위 결정**

- 재생성할 기간 결정
- 재생성할 과목 결정
- 보존할 플랜 결정 (완료된 플랜, 진행중인 플랜 등)

**3단계: 재생성 조건 설정**

- 기존 설정 유지 또는 새 설정 적용
- 학습 진행 상황 반영 여부
- 학습 관리 AI 제안 반영 여부

**4단계: 플랜 재생성**

- 플랜 생성 AI 실행
- 학습 진행 상황 반영
- 변경된 조건 반영
- 최적화 수행

**5단계: 비교 및 확인**

- 기존 플랜 vs 재생성 플랜 비교
- 변경 사항 요약 표시
- 사용자 확인 및 승인

**6단계: 적용**

- 재생성 플랜 적용
- 기존 플랜 백업 (히스토리 저장)
- 학습 기록 업데이트

#### 입력 데이터

**재생성 요청**

- 플랜 ID: 문자열 (필수)
- 재생성 범위: "전체" | "부분" (필수)
- 재생성 옵션: 객체 (선택)
  - 기간: 객체 (시작일, 종료일, 부분 재생성 시)
  - 과목: 과목 ID 배열 (부분 재생성 시)
  - 보존할 플랜: 플랜 ID 배열 (완료된 플랜 등)
  - 학습 진행 상황 반영: boolean
  - 학습 관리 AI 제안 반영: boolean

**변경된 조건** (선택)

- 학습 시간 설정: 객체 (변경된 경우)
- 자율 학습 시간 설정: 객체 (변경된 경우)
- 학습 제외일: 배열 (추가/삭제된 경우)
- 학원 일정: 배열 (변경된 경우)
- 학습 시간 제외 항목: 배열 (변경된 경우)
- 학습일/복습일 주기: 객체 (변경된 경우)

**학습 진행 상황** (자동 수집)

- 완료된 플랜: 플랜 ID 배열
- 진행중인 플랜: 플랜 ID 배열
- 미완료 플랜: 플랜 ID 배열
- 학습 기록: 객체 (학습 시간, 진행률 등)

#### 출력 데이터

**재생성 결과**

- 재생성된 플랜: 플랜 객체 배열
- 재생성 요약: 객체
  - 재생성된 플랜 수: 숫자
  - 삭제된 플랜 수: 숫자
  - 변경된 플랜 수: 숫자
  - 보존된 플랜 수: 숫자

**비교 결과** (재생성 전)

- 기존 플랜 요약: 객체 (플랜 수, 총 학습 시간 등)
- 재생성 플랜 요약: 객체 (플랜 수, 총 학습 시간 등)
- 변경 사항: 객체 배열 (변경된 플랜 정보)
- 변경 사항 요약: 텍스트

**재생성 히스토리**

- 재생성 일시: ISO 8601 타임스탬프
- 재생성 범위: 문자열
- 재생성 이유: 텍스트
- 재생성 전 플랜 백업: 플랜 객체 배열

#### UI/UX 요구사항

**재생성 요청 화면**

- 재생성 범위 선택 (전체/부분 라디오 버튼)

### 19. 타이머 기능

#### 개요

학습 플랜 실행 시 실제 학습 시간을 측정하고 추적하는 타이머 기능

#### 기능 요구사항

**필수 기능**

- 타이머 시작/일시정지/재개/종료
- 실시간 타이머 표시 (시:분:초)
- 계획된 시간과 실제 시간 비교
- 타이머 종료 시 자동 기록 저장
- 백그라운드 실행 지원 (앱 최소화 시에도 동작)

**선택 기능**

- 타이머 알림/알람 설정
- 타이머 사운드 효과
- 타이머 위젯 (모바일)
- 집중 모드 (다른 앱 차단)
- 휴식 시간 타이머

#### 타이머 데이터 구조

**타이머 상태**

- 상태: "idle" | "running" | "paused" | "completed"
- 시작 시간: ISO 8601 타임스탬프
- 일시정지 시간: ISO 8601 타임스탬프 (배열)
- 재개 시간: ISO 8601 타임스탬프 (배열)
- 종료 시간: ISO 8601 타임스탬프

**시간 정보**

- 계획된 시간: 숫자 (분)
- 경과 시간: 숫자 (초, 실시간 업데이트)
- 일시정지된 시간: 숫자 (초, 총합)
- 실제 학습 시간: 숫자 (초, 경과 시간 - 일시정지된 시간)

**설정 정보**

- 알림 설정: boolean
- 사운드 설정: boolean
- 집중 모드: boolean
- 휴식 시간: 숫자 (분, 선택)

#### 입력 데이터

- 플랜 ID: 문자열 (필수)
- 계획된 시간: 숫자 (분, 필수)
- 알림 설정: boolean (선택)
- 사운드 설정: boolean (선택)
- 집중 모드: boolean (선택)
- 휴식 시간: 숫자 (분, 선택)

#### 출력 데이터

- 타이머 상태: 객체 (현재 상태, 경과 시간 등)
- 타이머 기록: 객체 (완료된 타이머 정보)

#### UI/UX 요구사항

- 큰 숫자로 타이머 표시 (시:분:초)
- 시작/일시정지/재개/종료 버튼 명확히 표시
- 계획된 시간과 경과 시간 비교 표시
- 진행률 시각화 (원형 프로그레스 바 등)
- 알림/알람 시각적/청각적 피드백
- 다크 모드 지원
- 반응형 디자인 지원
- 모바일: 항상 켜짐 화면 옵션

#### 기술 요구사항

- 백그라운드 실행: Service Worker (웹) / Background Task (모바일)
- 정확한 시간 측정: 시스템 시간 기반
- 오프라인 지원: 로컬 저장 후 동기화
- 데이터 동기화: 실시간 또는 주기적 동기화

## 타임라인 캘린더 뷰

### 20. 타임라인 캘린더 뷰

#### 개요

학습 플랜을 시간대별로 시각화하여 월별, 주별, 일별로 조회할 수 있는 캘린더 뷰 기능

#### 기능 요구사항

**필수 기능**

- 월별 뷰: 한 달 단위로 플랜 조회
- 주별 뷰: 한 주 단위로 플랜 조회
- 일별 뷰: 하루 단위로 상세 플랜 조회
- 뷰 전환: 월/주/일 뷰 간 전환
- 날짜 네비게이션: 이전/다음 기간 이동
- 플랜 상세 정보 표시: 클릭/탭 시 상세 정보 모달/드로어
- 시간대별 플랜 배치: 실제 시간대에 맞춰 플랜 표시

**선택 기능**

- 오늘 날짜 하이라이트
- 플랜 상태별 색상 구분 (예정, 진행중, 완료, 미완료)
- 드래그 앤 드롭으로 플랜 시간 조정
- 플랜 필터링: 과목별, 상태별 필터
- 플랜 검색: 플랜명, 과목명으로 검색
- 일정 추가/수정/삭제 (인라인 편집)

#### 뷰별 상세 기능

**월별 뷰 (Month View)**

- 월 단위 캘린더 그리드 표시
- 날짜별 플랜 개수 표시
- 날짜별 총 학습 시간 표시
- 날짜별 완료율 표시 (프로그레스 인디케이터)
- 날짜 클릭 시 일별 뷰로 이동
- 주말/공휴일 구분 표시

**주별 뷰 (Week View)**

- 주 단위 타임라인 표시
- 요일별 컬럼 구성
- 시간대별 행 구성 (예: 30분 단위)
- 플랜을 시간대에 맞춰 블록으로 표시
- 플랜 블록에 과목명, 시간 표시
- 스크롤 가능한 타임라인
- 오늘 날짜 하이라이트

**일별 뷰 (Day View)**

- 하루 단위 상세 타임라인 표시
- 시간대별 세부 구성 (예: 15분 단위)
- 플랜 상세 정보 표시 (과목, 교재, 강의, 시간 등)
- 플랜 상태 표시 (예정, 진행중, 완료, 미완료)
- 플랜 시작/일시정지/완료 버튼
- 학습 시간 통계 표시 (일일 총 학습 시간, 완료율)

#### 데이터 구조

**캘린더 뷰 데이터**

- 현재 뷰 타입: "month" | "week" | "day"
- 현재 날짜 범위: 시작일 ~ 종료일
- 플랜 목록: 플랜 객체 배열
- 날짜별 통계: 객체 (플랜 개수, 총 학습 시간, 완료율)

**플랜 표시 정보**

- 플랜 ID: 고유 식별자
- 날짜: YYYY-MM-DD
- 시작 시간: HH:mm
- 종료 시간: HH:mm
- 소요 시간: 숫자 (분)
- 과목명: 문자열
- 과목 색상: HEX 코드
- 상태: "scheduled" | "in_progress" | "completed" | "skipped"
- 진행률: 숫자 (0-100)

#### 입력 데이터

- 뷰 타입: "month" | "week" | "day" (필수)
- 기준 날짜: YYYY-MM-DD (필수)
- 필터 옵션: 객체 (선택)
  - 과목 목록: 과목 ID 배열
  - 상태 목록: 상태 배열
  - 검색어: 문자열

#### 출력 데이터

- 캘린더 데이터: 객체
  - 날짜 범위: 시작일 ~ 종료일
  - 플랜 목록: 플랜 객체 배열
  - 날짜별 통계: 객체
- 플랜 상세: 단일 객체 (클릭 시)

#### UI/UX 요구사항

**공통**

- 반응형 디자인 지원 (모바일/태블릿/데스크톱)
- 다크 모드 지원
- 부드러운 애니메이션 전환
- 터치 제스처 지원 (스와이프, 핀치 줌)
- 접근성 지원 (키보드 네비게이션, 스크린 리더)

**월별 뷰**

- 그리드 레이아웃 (7일 × 주 수)
- 날짜 셀에 플랜 개수/시간 표시
- 날짜 셀 클릭 시 일별 뷰로 이동
- 월 네비게이션 (이전/다음 월)

**주별 뷰**

- 타임라인 레이아웃 (요일 × 시간대)
- 시간대 스크롤 가능
- 플랜 블록 드래그 앤 드롭 (선택)
- 주 네비게이션 (이전/다음 주)
- 오늘 날짜 하이라이트

**일별 뷰**

- 타임라인 레이아웃 (시간대 세로 배치)
- 플랜 상세 정보 표시
- 플랜 액션 버튼 (시작/일시정지/완료)
- 일 네비게이션 (이전/다음 일)
- 시간대별 통계 표시

**색상 및 시각화**

- 과목별 색상 구분
- 상태별 색상 구분
  - 예정: 회색
  - 진행중: 파란색
  - 완료: 초록색
  - 미완료: 빨간색
- 진행률 프로그레스 바
- 호버/포커스 효과

#### 기술 요구사항

- 가상 스크롤: 대량 데이터 효율적 렌더링
- 실시간 업데이트: WebSocket 또는 Polling
- 오프라인 지원: 로컬 캐싱
- 성능 최적화: 뷰포트 기반 렌더링
- 브라우저 호환성: 최신 브라우저 지원

### 21. 뷰별 표시 항목 설정

#### 개요

타임라인 캘린더 뷰에서 각 뷰(월/주/일)별로 표시할 항목을 사용자가 커스터마이징할 수 있는 기능

#### 기능 요구사항

**필수 기능**

- 뷰별 표시 항목 선택/해제
- 표시 항목 순서 변경 (드래그 앤 드롭)
- 설정 저장 및 불러오기
- 기본 설정 제공 및 초기화

**선택 기능**

- 표시 항목별 크기/폰트 크기 조정
- 표시 항목별 색상 커스터마이징
- 프로필별 설정 저장 (여러 설정 프로필 관리)

- 날짜 (필수, 비활성화 불가)
- 플랜 개수
- 총 학습 시간
- 완료율 (프로그레스 바)
- 완료된 플랜 개수
- 미완료 플랜 개수
- 과목별 플랜 개수
- 주말/공휴일 표시
- 오늘 날짜 하이라이트

**주별 뷰 표시 항목**

- 날짜/요일 (필수, 비활성화 불가)
- 시간대 (필수, 비활성화 불가)
- 플랜 블록 (필수, 비활성화 불가)
- 플랜 과목명
- 플랜 시간 (시작-종료)
- 플랜 소요 시간
- 플랜 상태 아이콘
- 플랜 진행률
- 플랜 교재/강의 정보
- 시간대별 통계

**일별 뷰 표시 항목**

- 날짜 (필수, 비활성화 불가)
- 시간대 (필수, 비활성화 불가)
- 플랜 상세 정보 (필수, 비활성화 불가)
- 플랜 과목명
- 플랜 교재명
- 플랜 강의명
- 플랜 시간 (시작-종료)
- 플랜 소요 시간
- 플랜 상태
- 플랜 진행률
- 플랜 메모
- 플랜 태그
- 일일 총 학습 시간
- 일일 완료율
- 시간대별 통계

#### 설정 데이터 구조

**뷰별 설정**

- 뷰 타입: "month" | "week" | "day"
- 표시 항목 목록: 표시 항목 ID 배열
- 표시 항목 순서: 표시 항목 ID 배열 (순서 포함)

  - 항목 ID: 문자열
  - 표시 여부: boolean
  - 크기: "small" | "medium" | "large"
  - 색상: HEX 코드 (선택)
  - 폰트 크기: 숫자 (선택)

**설정 프로필**

- 프로필 ID: 고유 식별자
- 프로필명: 문자열
- 뷰별 설정: 뷰별 설정 객체 배열
- 기본 프로필 여부: boolean
- 생성일: ISO 8601 타임스탬프
- 수정일: ISO 8601 타임스탬프

#### 입력 데이터

- 뷰 타입: "month" | "week" | "day" (필수)
- 표시 항목 목록: 표시 항목 ID 배열 (필수)
- 표시 항목 순서: 표시 항목 ID 배열 (필수)
- 표시 항목별 옵션: 객체 (선택)
- 프로필명: 문자열 (선택, 프로필 저장 시)

#### 출력 데이터

- 뷰별 설정: 객체 (현재 설정)
- 설정 프로필 목록: 배열 (저장된 프로필 목록)
- 설정 프로필 상세: 단일 객체 (프로필별 설정)

#### UI/UX 요구사항

**설정 화면**

- 뷰별 탭으로 구분 (월/주/일)
