## Executive Summary

### 제품 개요

**1730Timetable**은 학생의 성적 분석을 기반으로 맞춤형 학습 플랜을 자동 생성하는 AI 기반 통합 학습 관리 시스템입니다. 내신성적과 모의고사 성적을 분석하여 적합한 교재와 강의를 추천하고, 학생의 수준과 학습 가능 시간을 고려하여 최적의 학습 스케줄을 자동 생성합니다.

### 핵심 가치 제안

- **AI 기반 맞춤형 학습 플랜**: 학생 성적 분석을 통한 개인화된 학습 경로 제공
- **자동 스케줄링**: 복잡한 시간 제약 조건을 고려한 최적 학습 스케줄 자동 생성
- **실시간 모니터링**: 학습 진행 상황 실시간 추적 및 피드백
- **멀티테넌트 지원**: 여러 프렌차이즈 지점이 독립적으로 사용 가능

### 목표 시장

- **주요 고객**: 학원, 학습 관리 기관, 프렌차이즈 교육 기관
- **사용자**: 담당자(선생님), 학생, 학부모, 관리자
- **시장 규모**: 국내 사교육 시장 (연간 약 20조원 규모)

## Product Overview

### 제품 비전

"모든 학생이 자신에게 최적화된 학습 플랜으로 효율적으로 학습할 수 있도록 돕는다"

### 제품 미션

학생의 성적 데이터와 학습 패턴을 AI로 분석하여, 개인별 맞춤형 학습 플랜을 자동 생성하고, 실시간으로 학습 진행을 모니터링하여 학습 효과를 극대화하는 통합 학습 관리 플랫폼을 제공합니다.

### 핵심 기능

1. **성적 분석 기반 교재/강의 추천**

   - 내신성적과 모의고사 성적 분석
   - 학생 수준에 맞는 교재 및 강의 자동 추천

2. **자동 플랜 생성**

   - 학습 시간대, 제외 시간, 주기 등을 고려한 최적 스케줄 생성
   - 취약과목/전략과목별 차별화된 학습 분량 배정

3. **타임라인 캘린더 뷰**

   - 월별/주별/일별 학습 플랜 시각화
   - 시간대별 상세 플랜 조회 및 실행

4. **학습 진행 관리**

   - 플랜 실행, 학습 기록, 통계 및 리포트 제공
   - 실시간 학습 진행 상황 추적

5. **통합 플랜 관리**
   - 개별 플랜 및 통합 플랜(캠프 프로그램) 지원
   - 다수 학생 대상 일괄 플랜 생성

### 제품 특징

- **멀티테넌트 구조**: 여러 프렌차이즈 지점이 독립적으로 사용 가능
- **AI 기반 분석**: 학생 성적, 교재, 강의를 AI로 분석하여 맞춤형 추천
- **실시간 모니터링**: 학습 진행 상황 실시간 추적 및 피드백
- **확장 가능한 아키텍처**: 향후 기능 확장 및 외부 시스템 연동 지원

## Goals and Objectives

### 비즈니스 목표

#### 단기 목표 (6개월)

1. **MVP 출시**

   - 핵심 기능(플랜 생성, 학습 관리) 완성
   - 최소 3개 테넌트(프렌차이즈 지점) 온보딩
   - 사용자 만족도 80% 이상 달성

2. **사용자 확보**

   - 활성 담당자 50명 이상
   - 활성 학생 500명 이상
   - 일일 활성 사용자(DAU) 200명 이상

3. **기능 안정화**
   - 시스템 가동률 99% 이상
   - 플랜 생성 성공률 95% 이상
   - 평균 응답 시간 2초 이하

#### 중기 목표 (1년)

1. **시장 확대**

   - 테넌트 20개 이상 확보
   - 활성 학생 5,000명 이상
   - 월간 매출 1억원 이상

2. **기능 고도화**

   - AI 분석 정확도 향상 (추천 정확도 90% 이상)
   - 리포트 및 통계 시스템 구축
   - 알림 및 커뮤니케이션 시스템 구축

3. **사용자 경험 개선**
   - 사용자 만족도 90% 이상
   - 사용자 이탈률 10% 이하
   - NPS(Net Promoter Score) 50 이상

#### 장기 목표 (2년)

1. **시장 리더십**

   - 국내 학습 관리 시스템 시장 점유율 10% 이상
   - 테넌트 100개 이상 확보
   - 활성 학생 50,000명 이상

2. **기술 혁신**

   - 머신러닝 기반 예측 분석 도입
   - 외부 교육 플랫폼 연동 (EBS, 메가스터디 등)
   - 모바일 앱 출시

3. **수익성 확보**
   - 월간 매출 10억원 이상
   - EBITDA 양수 전환
   - 시리즈 A 투자 유치

## User Stories

### Epic 1: 플랜 생성 및 관리

#### Story 1.1: 성적 분석 기반 교재/강의 추천

**As a** 담당자  
**I want to** 학생의 내신성적과 모의고사 성적을 입력하고 AI 분석 결과를 확인  
**So that** 학생에게 적합한 교재와 강의를 추천할 수 있다

**Acceptance Criteria:**

- 내신성적과 모의고사 성적을 입력할 수 있다
- AI 분석 결과(학생 수준, 강점/약점, 전략과목/취약과목)를 확인할 수 있다
- 추천 교재와 강의 목록을 확인할 수 있다
- 추천 이유를 확인할 수 있다

#### Story 1.2: 자동 플랜 생성

**As a** 담당자  
**I want to** 학습 기간, 시간대, 제외일 등을 설정하고 플랜을 자동 생성  
**So that** 학생에게 최적화된 학습 스케줄을 제공할 수 있다

**Acceptance Criteria:**

- 플랜 기간(시작일, 종료일)을 설정할 수 있다
- 학습 시간대, 자율학습 시간, 제외일, 학원 일정 등을 설정할 수 있다
- 학습일/복습일 주기를 설정할 수 있다
- 플랜 생성 버튼을 클릭하면 자동으로 플랜이 생성된다
- 생성된 플랜을 타임라인 캘린더 뷰에서 확인할 수 있다

#### Story 1.3: 플랜 수정 및 재생성

**As a** 담당자  
**I want to** 생성된 플랜을 수정하거나 재생성  
**So that** 변경된 조건이나 학습 진행 상황을 반영할 수 있다

**Acceptance Criteria:**

- 생성된 플랜을 수정할 수 있다
- 플랜 재생성 기능을 사용할 수 있다
- 재생성 전 기존 플랜과 비교 미리보기를 확인할 수 있다
- 학습 진행 상황을 반영하여 재생성할 수 있다

### Epic 2: 학습 실행 및 기록

#### Story 2.1: 플랜 실행

**As a** 학생  
**I want to** 오늘의 학습 플랜을 확인하고 실행  
**So that** 계획된 학습을 진행할 수 있다

**Acceptance Criteria:**

- 오늘의 학습 플랜 목록을 확인할 수 있다
- 플랜을 클릭하여 상세 정보를 확인할 수 있다
- 플랜 시작 버튼을 클릭하여 학습을 시작할 수 있다
- 타이머가 시작되고 실시간으로 시간이 표시된다

#### Story 2.2: 학습 기록

**As a** 학생  
**I want to** 학습 진행 상황을 기록  
**So that** 학습 이력을 추적하고 통계를 확인할 수 있다

**Acceptance Criteria:**

- 학습 시작/일시정지/재개/완료 기능을 사용할 수 있다
- 실제 학습 시간이 자동으로 기록된다
- 학습 진행률이 자동으로 계산되어 표시된다
- 학습 기록을 일별/주별/월별로 조회할 수 있다

#### Story 2.3: 학습 통계 조회

**As a** 학생  
**I want to** 학습 통계를 확인  
**So that** 학습 효과를 객관적으로 확인할 수 있다

**Acceptance Criteria:**

- 총 학습 시간을 확인할 수 있다
- 학습 완료율을 확인할 수 있다
- 과목별 학습 시간 통계를 확인할 수 있다
- 학습 추이 그래프를 확인할 수 있다

### Epic 3: 리포트 및 통계

#### Story 3.1: 학습 리포트 생성

**As a** 담당자  
**I want to** 학생별 학습 리포트를 생성

- **AI 서비스**: OpenAI API (또는 자체 모델)
- **분석 엔진**: Python (선택)

### 아키텍처 요구사항

#### 멀티테넌트 구조

- **방식**: 테넌트 ID 기반 격리 (Shared Database)
- 모든 테이블에 `tenant_id` 컬럼 추가
- 모든 쿼리에 `WHERE tenant_id = ?` 자동 추가
- 애플리케이션 레벨에서 격리 보장

#### 데이터베이스 설계

- **주요 테이블**:
  - `tenants` (테넌트)
  - `users` (사용자)
  - `students` (학생)
  - `scores` (성적)
  - `textbooks` (교재)
  - `lectures` (강의)
  - `plans` (플랜)
  - `plan_items` (플랜 항목)
  - `study_sessions` (학습 세션)

#### API 설계

- **RESTful API** 설계
- **OpenAPI/Swagger** 명세서 작성
- **인증**: JWT 토큰 기반
- **Rate Limiting**: API 호출 제한

#### 보안 요구사항

- **HTTPS** 통신 필수
- **비밀번호 해시화**: bcrypt
- **SQL Injection 방지**: 파라미터화된 쿼리
- **XSS 방지**: 입력 데이터 검증 및 이스케이프
- **CSRF 방지**: CSRF 토큰 사용
- **세션 관리**: 안전한 세션 타임아웃 설정

#### 성능 요구사항

- **응답 시간**: 평균 2초 이하
- **동시 사용자**: 최소 1,000명 지원
- **데이터베이스 쿼리 최적화**: 인덱싱, 쿼리 최적화
- **캐싱 전략**: Redis 또는 메모리 캐싱

#### 확장성 요구사항

- **수평 확장**: 로드 밸런싱 지원
- **마이크로서비스 아키텍처**: 향후 서비스 분리 가능
- **이벤트 기반 아키텍처**: 비동기 이벤트 처리

### 용어 정의

- **테넌트**: 프렌차이즈 지점 또는 독립적인 조직 단위
- **플랜**: 학습 스케줄 및 계획
- **플랜 항목**: 플랜 내 개별 학습 항목
- **학습일**: 실제 학습을 진행하는 날
- **복습일**: 직전 학습일의 내용을 복습하는 날
- **제외일**: 학습이 제외되는 날 (휴가, 개인일정 등)
- **전략과목**: 성적 향상에 전략적으로 집중할 과목
- **취약과목**: 성적이 낮아 보완이 필요한 과목
- **학습 세션**: 실제 학습을 진행하는 세션
